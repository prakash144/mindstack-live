    ☁️ Cloud-Native Development in Go¶

    📌 Overview¶

    Cloud-native development focuses on building scalable, resilient, and efficient applications using modern cloud technologies like Docker, Kubernetes, and Serverless.

    🐳 Docker – Containerizing Go Apps¶

    ✅ Why Docker?¶

    ✔️ Ensures consistency across environments.
    ✔️ Lightweight and portable.
    ✔️ Simplifies dependency management.

    📂 Typical Dockerfile for a Go App¶

    # Base image
FROM golang:1.21 AS builder
WORKDIR /app

# Copy and build
COPY . .
RUN go mod tidy && go build -o app

# Create a minimal runtime image
FROM alpine:latest
WORKDIR /root/
COPY --from=builder /app/app .
CMD ["./app"]

    🚀 Commands¶

    # Build Docker image
docker build -t my-go-app .

# Run container
docker run -p 8080:8080 my-go-app

    ☸️ Kubernetes – Orchestrating Go Apps¶

    ✅ Why Kubernetes?¶

    ✔️ Auto-scaling and self-healing.
    ✔️ Efficient resource management.
    ✔️ Declarative infrastructure with YAML.

    📂 Sample Kubernetes Deployment¶

    apiVersion: apps/v1
kind: Deployment
metadata:
  name: go-app
spec:
  replicas: 3
  selector:
    matchLabels:
      app: go-app
  template:
    metadata:
      labels:
        app: go-app
    spec:
      containers:
      - name: go-app
        image: my-go-app:latest
        ports:
        - containerPort: 8080

    🚀 Commands¶

    # Apply deployment
kubectl apply -f deployment.yaml

# Check running pods
kubectl get pods

    ⚡ Serverless – Go in FaaS (Function as a Service)¶

    ✅ Why Serverless?¶

    ✔️ Pay-per-execution model.
    ✔️ No infrastructure management.
    ✔️ Scales automatically.

    🚀 Running Go Serverless with AWS Lambda¶

    • Write a Go function using AWS Lambda runtime.
    • Package & deploy using AWS SAM or Serverless Framework.
    package main

import (
    "context"
    "github.com/aws/aws-lambda-go/lambda"
)

func handler(ctx context.Context) (string, error) {
    return "Hello from Go Lambda!", nil
}

func main() {
    lambda.Start(handler)
}

    🚀 Deploying¶

    # Build for AWS Lambda
GOOS=linux GOARCH=amd64 go build -o main

# Zip and upload
zip function.zip main
aws lambda create-function --function-name goLambda --zip-file fileb://function.zip --handler main --runtime go1.x

    ✅ Best Practices¶

    ✔️ Keep Docker images lightweight.
    ✔️ Use multi-stage builds to reduce image size.
    ✔️ Define resource limits in Kubernetes for optimal performance.
    ✔️ Optimize cold start time for serverless functions.
    ✔️ Use environment variables for config management.
