    üîÑ Inversion of Control (IoC)¶

    üìå Definition¶

    • A design principle where the control of object creation and dependency management is transferred from the application code to the Spring container.
    • Promotes loose coupling and better testability.

    ‚öôÔ∏è How it works in Spring¶

    • The Spring IoC Container is responsible for:

      • Configuring beans
      • Wiring dependencies

    üß© Dependency Injection (DI)¶

    üìå Definition¶

    • A design pattern where the dependencies of an object are injected externally rather than being created internally.
    • Implements IoC.

    üí° Types of DI in Spring¶

    • Constructor Injection (preferred for mandatory dependencies)
    • Setter Injection (flexible, supports optional dependencies)
    • Field Injection (discouraged in production ‚Äì harder to test & less explicit)

    ‚öôÔ∏è Spring Annotations for DI¶

    • @Autowired ‚Äì automatically injects dependencies
    • @Qualifier ‚Äì used with @Autowired to resolve ambiguity
    • @Inject, @Resource ‚Äì JSR-330 & JSR-250 annotations (optional alternatives)

    ‚ö†Ô∏è Best Practices¶

    • Use constructor injection for required dependencies
    • Avoid field injection for testability and immutability
    • Keep configuration centralized and explicit (e.g., Java config classes or profiles)
