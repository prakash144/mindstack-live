    üîê What is Durability?¶

    Durability is the prevention of data loss. Amazon S3 offers: - 11 nines of durability (99.999999999%) - Statistically: 1 object out of 10,000 could be lost in 10 million years

    üß± 1. Data Redundancy¶

    üß† Key Idea: Mechanical HDDs are cheap but fail often ‚Üí replicate + shard data¶

    • Replication: Data is duplicated across multiple disks to ensure backup on failure.
    • Erasure Coding:
      • Splits data into data shards + parity shards
      • Original data is reconstructible from a subset of shards
      • Enhances durability with flexible replication
    • Shard Distribution: Shards are stored across many disks
    • Failure Recovery:
      • Background processes monitor disk health
      • Failed disks are quickly replaced
      • Parallelization allows faster recovery (free space on each device helps)

    ‚úÖ 2. Data Integrity¶

    üß† Key Idea: Prevent data corruption via checksums and transformation validation¶

    • Checksum on Upload:
      • Amazon client SDK adds checksum (CRC32C, SHA-1)
      • Detects bit flips in transmission
    • Checksum in HTTP Trailer: Efficient validation without re-reading whole data
    • Checksum on Storage:
      • Erasure-coded shards also get checksums
      • Enables validation during reads
    • Bracketing:
      • Reverse all transformations before confirming write
      • Validates that uploaded data = stored data

    üß™ 3. Data Audit¶

    üß† Key Idea: Failures are unpredictable ‚Üí scan and re-replicate frequently¶

    • Hard Disk Monitoring:
      • Detect failure and scale repair services
      • Includes bad sectors (bit flips on disk, not entire disk failure)
    • Periodic Audits:
      • Use stored checksums to validate erasure-coded shards
      • Re-replicate on detection of corruption

    üß© 4. Data Isolation¶

    üß† Key Idea: Separate concerns to minimize blast radius¶

    • Metadata and Content:
      • Metadata ‚Üí stored in databases
      • File content ‚Üí stored in disk arrays
    • Availability Zones (AZ):
      • Physically separated, isolated power and network
      • Prevents simultaneous failures
    • User Errors:
      • Versioning: Tracks changes with version IDs; delete = adding a delete marker
      • Object Lock: Prevents deletion for a specified period
      • Backups: Point-in-time recovery for safety

    üß† 5. Engineering Culture¶

    üß† Key Idea: Durability is part of the architecture and mindset¶

    • Durability Reviews:
      • Mandatory before every change
      • Includes threat models and simplification plans
    • Written Documentation:
      • Risks are documented and assessed
    • Mathematical Modeling:
      • Proven models used to calculate durability impact
    • Philosophy: Durability is a continuous process

    üß¨ Erasure Coding¶

    • Recoverable from subset of parts
    • Higher efficiency and durability than basic replication

    üßÆ Checksums (CRC32C, SHA-1)¶
