    💻 Technical – Backend Interview Mock Test (8+ YOE)¶

    ⭐⭐ 1. Explain how you design a REST API. What best practices do you follow?¶

    • 📝 Start with Business Requirements: Understand core functionality, edge cases, expected usage.
    • 📦 Resource-Oriented Endpoints: Use nouns for URIs (e.g., /users, /orders).
    • 🔁 HTTP Methods: Follow RESTful verbs - GET, POST, PUT/PATCH, DELETE.
    • 📌 Versioning: Use URI versioning like /api/v1/... to maintain backward compatibility.
    • 📟 Status Codes: Return proper HTTP status codes (200, 400, 401, 403, 500).
    • 🔍 Pagination & Filtering: Enable for large lists using query params (?page=1&limit=20).
    • 🚨 Structured Error Handling: Standardize errors with code, message, and field.
    • 📄 API Documentation: Use OpenAPI/Swagger for collaboration and clarity.

    🔧 Real Use Case: At Delhivery, designed billing APIs with versioning and robust filtering, enabling scalable invoice generation.

    ⭐⭐ 2. What’s your approach to securing APIs?¶

    • 🔐 Authentication: Implement JWT tokens with expiry and refresh token strategies.
    • 🛡️ Authorization (RBAC): Enforce endpoint-level role-based access policies.
    • 📡 HTTPS Only: Encrypt all data in transit.
    • 🚫 Rate Limiting: Throttle requests via API gateway to prevent abuse.
    • 🧼 Input Validation: Sanitize and validate all inputs at multiple layers.
    • 🧾 Audit Logging: Log suspicious requests and authentication failures.

    🛠 Example: At Thales Gambit, used OAuth2 + JWT + key manager policy mapping for layered API security.

    ⭐⭐ 3. Can you walk us through how you would implement authentication and RBAC in a backend system?¶

    • 🔑 JWT Authentication: Issue token post-login with user context, signed and time-bound.

    • 🧭 RBAC Enforcement:

      • Define roles (Admin, Viewer, etc.)
      • Map roles to permissions
      • Use middleware/decorators to enforce access checks.
      • 🗂️ Policy Store: Maintain centralized policy-to-role mapping in a config store or DB.

    ⚙️ Example: Used API Gateway with RBAC middleware in Thales Gambit to secure sensitive endpoints.

    ⭐⭐ 4. If given a Java (or Node.js/Python/Go) service that crashes under load, how would you troubleshoot and optimize it?¶

    • 🧪 Load Testing: Use JMeter, k6, or Locust to simulate production-like traffic.
    • 📊 Analyze Logs and Metrics: CPU, memory, heap, GC logs, thread dumps.
    • 🧠 Profiling: Tools like VisualVM, Py-Spy, Go pprof to locate bottlenecks.

    • 🔄 Optimizations:

      • Refactor tight loops / memory leaks
      • Minimize DB round-trips
      • Introduce caching and async calls
      • Tune server threads / scale horizontally
      • ♻️ Validation: Rerun performance tests post-fix to verify improvements.

    🎯 Real Case: At Delhivery, converted DB-heavy operations into async Redis-backed queues reducing response time by 40%.

    ⭐⭐ 5. How do you ensure that your backend services are scalable and maintainable in the long term?¶

    • 🧱 Microservices: Independent deployments, fault isolation, horizontal scaling.
    • 🧭 Service Discovery: Dynamically route service calls in distributed systems.
    • ⚖️ Stateless Design: Enables autoscaling and better fault recovery.
    • 🧩 DB Sharding: Distributes load across partitions.
    • 📬 Asynchronous Processing: Decouple with Kafka or RabbitMQ.
    • 🧪 CI/CD Pipelines: Automate testing, deployment, and rollback.

    📌 Example: At Delhivery, helped migrate billing system to microservices with load-balanced sharding support.

    ⭐ 6. Can you explain your approach to handling large datasets and ensuring optimal performance?¶

    • 🧮 Indexing: Speed up frequent queries with well-planned indexes.
    • ⚡ Caching: Redis/Memcached for hot data and repeated queries.
    • 📦 Batch Processing: Avoid long-running ops with chunked execution.
    • 🗃️ Partitioning: Split data (e.g., by date or region) for performance.
    • 🔄 ETL Pipelines: Extract, transform, and load data efficiently.

    📊 Example: At Delhivery, used Redis and time-based partitioning to enhance analytics query speed by 60%.

    ⭐ 7. How do you handle error logging and monitoring in production?¶

    • 📚 Centralized Logging: ELK, Splunk, or Cloud-native tools (e.g., GCP Logging).
    • 🚦 Log Levels: Categorize logs (INFO, WARN, ERROR, FATAL).
    • 📢 Alerting: Alertmanager, PagerDuty, or CloudWatch Alarms for critical incidents.
    • ❤️ Health Checks: Integrate liveness/readiness probes with infra orchestration.
    • 🧹 Retention & Rotation: Efficient storage by archiving and cleaning old logs.

    📌 Example: At Thales, we used Kibana dashboards and proactive alerting to monitor system health.

    💡 Additional Questions You Should Know¶

    🔹 8. How do you approach testing in your backend services?¶

    • ✅ Unit Testing: Mock external services and isolate logic.
    • 🔄 Integration Testing: Test modules/services together with real dependencies.
    • 📌 TDD: Write test cases before writing production code.
    • 🛠️ Frameworks: JUnit (Java), PyTest (Python), Mocha (Node.js).

    🔹 9. What strategies do you use to handle high concurrency in backend systems?¶

    • 🧵 Thread Pools / Async Handling: Efficient thread/resource usage.
    • 📊 Optimized DB Transactions: Minimize locking and deadlocks.
    • 🌐 Load Balancing: Distribute requests across replicas.
    • ⚖️ Horizontal Scaling: Scale instances with autoscaling groups.
