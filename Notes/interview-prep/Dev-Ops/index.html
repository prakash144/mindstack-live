    ☁️ DevOps / Cloud / CI-CD Mock Interview for 8+ Years Experience¶

    🌟🌟🌟 1. What’s your experience with Docker? Can you explain how you'd containerize a web application?¶

    🧠 Key Points:¶

    • 🐳 Extensive experience containerizing frontend and backend apps using Docker.
    • 🛠️ Steps to containerize a web app:
      • 📄 Write a Dockerfile:
        • Choose base image (e.g., node:18-alpine for Node.js).
        • Copy application source code.
        • Install dependencies.
        • Expose required ports.
        • Define CMD to run app.
      • 🔨 Build image using: bash docker build -t my-app .
      • ▶️ Run container using: bash docker run -p 3000:3000 my-app

    🌟🌟🌟 2. How would you set up a CI/CD pipeline in GitLab for a full-stack application?¶

    🧠 Key Points:¶

    • 📄 Use .gitlab-ci.yml to define pipeline stages: install, build, test, deploy.
    • 💼 Split jobs:
      • frontend-build: Build/test frontend.
      • backend-build: Build/test backend.
    • 🐳 Use Docker runners for environment consistency.
    • ⚡ Cache dependencies to improve build speed.
    • ✅ Run tests using commands like npm test or pytest.
    • 🚀 Automate deployment to staging/production (can integrate with Kubernetes).
    • 🔐 Manage secrets using GitLab CI/CD environment variables.

    🌟🌟 3. What are the basic components of Kubernetes you need to deploy a service?¶

    🧠 Key Components:¶

    • 📦 Pod – Smallest deployable unit, contains container(s).
    • 🧩 Deployment – Handles replica sets, scaling, and updates.
    • 🌐 Service – Exposes Pods (types: ClusterIP, NodePort, LoadBalancer).
    • ⚙️ ConfigMap & Secrets – For non-sensitive and sensitive configurations respectively.

    🌟🌟 4. What experience do you have with Azure, and how would you provision a basic virtual machine or storage account using the Azure CLI or portal?¶

    🧠 Key Points:¶

    • 💻 Experience with hosting apps, managing VMs, and provisioning resources on Azure.
    • ⚙️ Azure CLI Examples:
      • Provision VM: bash az vm create --resource-group myGroup --name myVM --image UbuntuLTS --generate-ssh-keys
      • Provision storage account: bash az storage account create --name mystorage --resource-group myGroup --location eastus --sku Standard_LRS
    • 🖱️ Azure Portal: GUI-based provisioning via web interface.

    🌟🌟 5. Can you explain the difference between Continuous Integration, Continuous Delivery, and Continuous Deployment?¶

    🧠 Key Concepts:¶

    • 🔁 CI (Continuous Integration):
      • Frequent code merges.
      • Automated build & test validation.
    • 🚀 CD (Continuous Delivery):
      • Code is always production-ready.
      • Auto-deploy to staging.
    • ⚙️ CD (Continuous Deployment):
      • Auto-deploy to production after tests pass.
      • Enables fast, reliable releases.

    🌟🌟 6. How would you monitor and troubleshoot a Kubernetes cluster?¶

    🧠 Key Tools & Methods:¶

    • 🔍 Use kubectl for basic diagnostics:
      • kubectl get pods
      • kubectl logs <pod>
      • kubectl describe <resource>
    • 📊 Prometheus + Grafana for metric collection and dashboards.
    • 🔁 Kube-state-metrics for resource-level insights.
    • 🚨 Set up alerting with Alertmanager for anomaly detection.

    🌟🌟 7. How do you manage infrastructure as code? Can you give an example using Terraform or Azure Resource Manager?¶

    🧠 Key Points:¶

    • 🏗️ Manage infrastructure using Terraform and ARM templates.
    • 📄 Terraform steps:
      • Define .tf files for resources (VMs, storage, etc.).
      • Run: bash terraform plan terraform apply
      • Sample VM resource: hcl resource "azurerm_virtual_machine" "example" { name = "example-vm" location = "East US" resource_group_name = azurerm_resource_group.example.name ... }
    • 🔄 Benefits: reproducibility, version control, easy updates.

    🌟🌟 8. Can you explain the concept of Blue-Green Deployments? How does it work in a CI/CD pipeline?¶

    🧠 Key Points:¶

    • 🎯 Deploy to Green (new) environment while Blue (current) serves traffic.
    • 🔄 After verification, switch traffic to Green.
    • 🔙 Blue acts as fallback in case of failure.
    • ✅ In pipeline:
      1. Deploy to Green.
      2. Test.
      3. Shift traffic.
      4. Optionally decommission Blue.

    🌟🌟 9. How would you secure sensitive data (like API keys, passwords) in a CI/CD pipeline?¶

    🧠 Key Practices:¶

    • 🔐 Use environment variables and encrypted secrets in GitLab/GitHub CI.
    • 🧰 Use secret managers like HashiCorp Vault, AWS Secrets Manager, or Azure Key Vault.
    • 🧷 Encrypt data in transit & at rest.
    • 👥 Implement strict RBAC for secret access.

    🌟🌟 10. What’s your experience with monitoring and logging for cloud applications?¶

    🧠 Key Tools:¶

    • 🔭 Monitoring:
      • CloudWatch (AWS), Azure Monitor, Prometheus, Grafana.
    • 📝 Logging:
      • Use frameworks like Winston (Node.js) to send logs to ELK or CloudWatch Logs.
    • 🚨 Alerting based on key metrics: CPU, memory, response time, error rates.
