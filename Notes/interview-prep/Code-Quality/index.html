    🧪 Testing & Code Quality - Mock Interview Questions¶

    ⭐⭐ 1. ✅ What is Test-Driven Development (TDD), and how have you applied it in your past work?¶

    • 🧠 TDD is a development approach where tests are written before writing production code.
    • 🔁 Follows the Red-Green-Refactor cycle:
      • ❌ Write a failing test
      • ✅ Write just enough code to pass the test
      • ♻ Refactor the code for maintainability
    • 💼 With 8+ years of experience, I’ve:
      • ⛓️ Applied TDD in microservice-based backends using PyTest and Jest
      • 🔍 Reduced production bugs by proactively defining edge cases
      • 🧩 Improved code modularity, enabling better collaboration during team development

    ⭐⭐ 2. 📊 What kind of tests do you usually write? Can you provide an example?¶

    • 🧪 Unit Tests: For pure logic validation (e.g., validators, parsers)
    • 🔗 Integration Tests: Ensure service-to-service or API-DB connectivity works as expected
    • 🎯 End-to-End Tests: Simulate user journeys in the actual system
    • 🛠️ Tools I’ve used across different layers:
      • ✅ Jest for JS/TS logic
      • 🔍 Supertest for API testing in Node.js
      • 🧪 Cypress for full UI regression
      • 🧬 Postman and Newman for API automation in CI

    ⭐ 3. 🤖 What are the principles of clean code you follow in your work?¶

    • 🧾 Use clear, expressive naming for variables, functions, and classes
    • 🔁 Apply DRY and KISS principles to avoid redundancy and over-engineering
    • 📏 Ensure single-responsibility methods that are small and testable
    • ⚖ Maintain consistency in coding style and patterns across the codebase
    • ❌ Avoid magic numbers, nested complexity, and over-commenting
    • ✅ Emphasize self-documenting code through meaningful structure and naming

    ⭐⭐ 4. 🗐️ How do you handle code reviews – both giving and receiving feedback?¶

    • 🎁 While giving:

      • 🧠 Focus on logic correctness, readability, and test coverage
      • 🤝 Encourage discussion instead of pushing personal preferences
      • 📌 Suggest refactors with reasoning and examples

    • 🙌 While receiving:

      • 🧘‍♂️ Stay open-minded and treat reviews as a learning opportunity
      • 🔍 Ask clarifying questions instead of assuming
      • 📈 Apply feedback promptly and acknowledge contributions

    ➕ Additional Questions You Should Prepare For:¶

    ⭐ 5. 🎡 What tools and frameworks do you use for test automation?¶

    • 💻 Backend:
      • Jest, Mocha → JS/TS unit testing
      • PyTest, unittest → Python logic testing
    • 🌐 API Testing:
      • Postman, Newman
    • 🖥️ Frontend:
      • Cypress, Playwright → E2E testing
    • ⚙ CI Integration:
      • Included test jobs in GitLab CI, GitHub Actions, and Jenkins pipelines

    ⭐⭐ 6. 🌎 How do you ensure your tests run efficiently in CI/CD pipelines?¶

    • ⚡ Prioritize unit tests early in the pipeline for fast feedback
    • 🧱 Use test stages – unit → integration → e2e
    • 🧊 Dockerized tests for consistent and isolated test environments
    • 🚀 Use parallel test execution and dependency caching to speed up builds
    • 🔍 Trigger targeted test suites based on code diff using tagging or paths

    ⭐ 7. 📊 How do you measure test coverage and quality?¶

    • 🛠️ Tools I’ve worked with:
      • Istanbul (JS/TS), coverage.py (Python), SonarQube for code quality
    • 📈 Key Metrics:
      • 🔍 Line and branch coverage %
      • 🧬 Mutation testing to assess test effectiveness
      • 🚨 Coverage thresholds enforced in CI for quality gates

    ⭐ 8. ❎ Can you write a pseudo test case for a simple function?¶

    Function: calculate_discount(price, is_premium)

    Test Case 1:
  - Input: price = 100, is_premium = true
  - Expected Output: 90

Test Case 2:
  - Input: price = 100, is_premium = false
  - Expected Output: 95

    💡 Final Tip¶

    • 🚨 Be ready with real bug stories or incidents where testing caught critical issues
    • ⚡ Emphasize how your testing mindset contributed to early detection, faster delivery, or better developer velocity
