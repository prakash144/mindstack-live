    🚀 CI/CD Pipeline for Go Applications¶

    📌 Overview¶

    Continuous Integration and Continuous Deployment (CI/CD) automates testing, building, and deploying Go applications, ensuring faster and reliable releases.

    🏗️ CI/CD Workflow¶

    1️⃣ Code Commit   → Developer pushes code to Git repository
2️⃣ Lint & Format → Run `gofmt`, `golangci-lint` to check code quality
3️⃣ Unit Tests    → Execute `go test ./...` for automated testing
4️⃣ Build        → Compile Go binaries for the target environment
5️⃣ Integration Tests → Run API & database tests
6️⃣ Docker Build  → Create a container image
7️⃣ Security Scan → Scan dependencies & image vulnerabilities
8️⃣ Deploy       → Push the image & deploy to Kubernetes/Cloud

    🛠️ Setting Up CI/CD with GitHub Actions¶

    📂 Directory Structure¶

    📦 project-name/
 ┣ 📂 .github/workflows/   # CI/CD pipeline definitions
 ┣ 📜 Dockerfile           # Containerization setup
 ┣ 📜 Makefile             # Build automation
 ┣ 📜 go.mod               # Dependency management
 ┣ 📜 go.sum               # Dependency checksums

    📝 Sample GitHub Actions Workflow (.github/workflows/ci.yml)¶

    name: Go CI/CD Pipeline

on: [push, pull_request]

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout Code
        uses: actions/checkout@v3

      - name: Set up Go
        uses: actions/setup-go@v4
        with:
          go-version: 1.21

      - name: Install Dependencies
        run: go mod tidy

      - name: Lint Code
        run: golangci-lint run ./...

      - name: Run Tests
        run: go test ./... -v

      - name: Build Application
        run: go build -o app ./...

      - name: Build Docker Image
        run: docker build -t myapp:latest .

      - name: Push to Docker Hub (Optional)
        run: |
          echo "${{ secrets.DOCKER_PASSWORD }}" | docker login -u "${{ secrets.DOCKER_USERNAME }}" --password-stdin
          docker tag myapp:latest myrepo/myapp:latest
          docker push myrepo/myapp:latest

      - name: Deploy to Kubernetes (Optional)
        run: kubectl apply -f k8s/deployment.yaml

    ✅ Best Practices¶

    ✔️ Use golangci-lint for static analysis.
    ✔️ Run tests before building the app.
    ✔️ Automate builds using Makefile.
    ✔️ Use Docker for containerized deployments.
    ✔️ Store secrets securely (e.g., GitHub Secrets, AWS Secrets Manager).
    ✔️ Automate deployment with Kubernetes, Terraform, or Ansible.
