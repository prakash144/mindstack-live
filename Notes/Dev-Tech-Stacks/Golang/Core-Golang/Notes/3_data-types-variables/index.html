    üî¢ Data Types & Variables in Go¶

    üìå Overview¶

    Go is a statically typed language with a rich set of built-in data types. Variables can be declared explicitly or inferred using type inference.

    üè∑Ô∏è Variable Declaration¶

    var name string = "John"  // Explicit type declaration
var age int = 30          // Integer type
var pi float64 = 3.14     // Float type
var isActive bool = true  // Boolean type

    üîπ Short Declaration (Type Inference)¶

    name := "Alice"   // Compiler infers type (string)
age := 25         // Compiler infers type (int)
isAdmin := false  // Compiler infers type (bool)

    üîπ Multiple Variable Declaration¶

    var (
    firstName, lastName string = "John", "Doe"
    age, height int = 30, 180
)

    üõ†Ô∏è Data Types in Go¶

    üî¢ Numeric Types¶

    Type Description Example
    int Signed integer (platform-dependent) var x int = 100
    int8 - int64 Fixed-size signed integers var y int8 = 127
    uint8 - uint64 Fixed-size unsigned integers var z uint16 = 65535
    float32, float64 Floating-point numbers var pi float64 = 3.14
    complex64, complex128 Complex numbers var c complex128 = 1 + 2i

    üî† String Type¶

    var message string = "Hello, Go!"
fmt.Println(len(message)) // Get string length
fmt.Println(message[0])   // Access character (returns byte)

    üîò Boolean Type¶

    var isGoFun bool = true
fmt.Println(isGoFun) // Output: true

    üèóÔ∏è Composite Types¶

    Type Description
    array Fixed-length collection of elements
    slice Dynamic-size collection (more common than arrays)
    map Key-value store (like dictionaries in Python)
    struct Custom data structure (like objects in OOP)
    interface Defines behavior for multiple types

    üîπ Example of Composite Types¶

    // Array (fixed size)
var numbers [3]int = [3]int{1, 2, 3}

// Slice (dynamic size)
names := []string{"Alice", "Bob", "Charlie"}

// Map (key-value pair)
user := map[string]int{"age": 30, "height": 180}

// Struct (custom type)
type Person struct {
    Name string
    Age  int
}
var p = Person{"John", 30}

    ‚úÖ Best Practices¶

    ‚úîÔ∏è Use := for concise variable declarations.
    ‚úîÔ∏è Prefer slice over array for dynamic collections.
    ‚úîÔ∏è Use map for fast lookups.
    ‚úîÔ∏è Leverage structs for organizing complex data.
    ‚úîÔ∏è Use explicit types where clarity is needed.
