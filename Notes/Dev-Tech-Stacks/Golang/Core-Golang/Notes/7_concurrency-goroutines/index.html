    ‚ö° Concurrency & Goroutines in Go¶

    üìå Overview¶

    Go is designed for concurrency with goroutines and channels, making it efficient for handling multiple tasks in parallel.

    üöÄ Goroutines ‚Äì Lightweight Threads¶

    • Goroutines are functions that run concurrently.
    • They are lighter than OS threads and managed by the Go runtime.
    • Use go keyword to start a goroutine.

    ‚úÖ Example:¶

    package main

import (
    "fmt"
    "time"
)

func printMessage(msg string) {
    for i := 0; i < 5; i++ {
        fmt.Println(msg, i)
        time.Sleep(time.Millisecond * 500)
    }
}

func main() {
    go printMessage("Goroutine")  // Runs concurrently
    printMessage("Main Function") // Runs in main thread
}

    ‚ö° Key Points:¶

    ‚úîÔ∏è Use go keyword to start a new goroutine.
    ‚úîÔ∏è Goroutines share the same memory space (use synchronization mechanisms if needed).
    ‚úîÔ∏è The main function does not wait for goroutines to finish by default.

    üîÑ Synchronization Techniques¶

    üõ†Ô∏è 1. WaitGroups (sync.WaitGroup)¶

    import (
    "fmt"
    "sync"
)

var wg sync.WaitGroup

func worker(id int) {
    defer wg.Done() // Decrement counter when done
    fmt.Println("Worker", id, "started")
}

func main() {
    wg.Add(2) // Set number of goroutines to wait for
    go worker(1)
    go worker(2)
    wg.Wait() // Block until all goroutines finish
    fmt.Println("All workers finished")
}

    üîÑ 2. Channels ‚Äì Safe Communication¶

    Channels help goroutines communicate safely.

    ch := make(chan string) // Create a channel
ch <- "Hello"           // Send data to channel
msg := <-ch             // Receive data from channel

    üîÅ Example: Sending Data Between Goroutines¶

    package main

import "fmt"

func sendMessage(ch chan string) {
    ch <- "Hello from Goroutine!"
}

func main() {
    ch := make(chan string)
    go sendMessage(ch)
    fmt.Println(<-ch) // Receive message
}

    üî• Best Practices¶

    ‚úîÔ∏è Use WaitGroups to ensure all goroutines complete before exiting.
    ‚úîÔ∏è Prefer channels over shared memory for communication.
    ‚úîÔ∏è Avoid goroutine leaks by properly closing channels.
    ‚úîÔ∏è Use context.Context for managing goroutine lifecycle.
