    ‚úÖ Why Docker?¶

    ‚úîÔ∏è Ensures consistency across environments.
    ‚úîÔ∏è Lightweight and portable.
    ‚úîÔ∏è Simplifies dependency management.

    üìÇ Typical Dockerfile for a Go App¶

# Copy and build

WORKDIR /root/
COPY --from=builder /app/app .

    ‚ò∏Ô∏è Kubernetes ‚Äì Orchestrating Go Apps¶

    ‚úÖ Why Kubernetes?¶

    ‚úîÔ∏è Auto-scaling and self-healing.
    ‚úîÔ∏è Efficient resource management.
    ‚úîÔ∏è Declarative infrastructure with YAML.

    üìÇ Sample Kubernetes Deployment¶

kind: Deployment
metadata:
  name: go-app
spec:
  replicas: 3
  selector:
    matchLabels:
      app: go-app
  template:
    metadata:
      labels:
        app: go-app
    spec:
      containers:
      - name: go-app
        image: my-go-app:latest
        ports:
        - containerPort: 8080

    üöÄ Commands¶

    ‚úÖ Why Serverless?¶

    ‚úîÔ∏è Scales automatically.

    üöÄ Running Go Serverless with AWS Lambda¶

    • Write a Go function using AWS Lambda runtime.
    • Package & deploy using AWS SAM or Serverless Framework.

    "context"
    "github.com/aws/aws-lambda-go/lambda"
)

    return "Hello from Go Lambda!", nil
}

    lambda.Start(handler)
