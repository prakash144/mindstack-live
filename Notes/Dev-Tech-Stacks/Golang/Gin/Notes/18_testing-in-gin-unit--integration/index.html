    🧪 Testing in Gin (Unit & Integration) 🚀¶

    📌 Introduction¶

    Testing is crucial for ensuring the reliability of Gin applications. Gin supports both unit tests (for handlers and middleware) and integration tests (for full API flows). We use Go’s built-in testing package and httptest for HTTP testing.

    🔹 Setting Up Testing in Gin¶

    ➤ Import Required Packages¶

    import (
    "net/http"
    "net/http/httptest"
    "testing"
    "github.com/gin-gonic/gin"
    "github.com/stretchr/testify/assert"
)

    ➤ Create a Testable Router¶

    func setupRouter() *gin.Engine {
    r := gin.Default()
    r.GET("/ping", func(c *gin.Context) {
        c.JSON(http.StatusOK, gin.H{"message": "pong"})
    })
    return r
}

    ✅ Unit Testing 🧩¶

    Unit tests focus on testing individual handlers without external dependencies.

    ➤ Basic Unit Test Example¶

    func TestPingRoute(t *testing.T) {
    router := setupRouter()
    w := httptest.NewRecorder()
    req, _ := http.NewRequest("GET", "/ping", nil)

    router.ServeHTTP(w, req)

    assert.Equal(t, http.StatusOK, w.Code)
    assert.JSONEq(t, `{"message":"pong"}`, w.Body.String())
}

    🔄 Integration Testing 🌐¶

    Integration tests validate multiple components working together, such as database and middleware.

    ➤ Example: Testing an API with Dependency Injection¶

    func TestUserEndpoint(t *testing.T) {
    router := gin.Default()
    router.GET("/user/:id", func(c *gin.Context) {
        id := c.Param("id")
        c.JSON(http.StatusOK, gin.H{"user_id": id})
    })

    w := httptest.NewRecorder()
    req, _ := http.NewRequest("GET", "/user/123", nil)
    router.ServeHTTP(w, req)

    assert.Equal(t, http.StatusOK, w.Code)
    assert.JSONEq(t, `{"user_id":"123"}`, w.Body.String())
}

    🛠️ Best Practices 🏆¶

    ✅ Use httptest for simulating requests. ✅ Use testify/assert for better readability. ✅ Separate unit and integration tests. ✅ Mock external dependencies (e.g., databases, services). ✅ Run tests with go test ./... for full coverage.

    🚀 Running the Tests¶

    go test -v

    This structured approach ensures a robust and maintainable Gin application. 🎯 Happy Testing! ✅
