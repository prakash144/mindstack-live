    üîÑ Basic Sorting¶

    üìù .sort() Method¶

    • ‚ú® Basic Usage: python elements = [5, 3, 6, 2, 1] elements.sort() # [1, 2, 3, 5, 6]
    • üîë Key Features:
      • Modifies list in-place
      • Returns None
      • Default ascending order

    üî§ String Sorting¶

    words = ["grape", "apple", "banana", "orange"]
words.sort()  # ['apple', 'banana', 'grape', 'orange']
    • ‚ö° Uses lexicographical order
    • Case-sensitive by default

    ‚öôÔ∏è Method Parameters¶

    üîß Function Signature¶

    def sort(key=None, reverse=False) -> None

    üîÑ Reverse Sorting¶

    • üìù Using reverse parameter: python elements = [5, 3, 6, 2, 1] elements.sort(reverse=True) # [6, 5, 3, 2, 1]
    • üîÑ Alternative approach: python elements.sort() elements.reverse()

    üéØ Custom Sorting¶

    üîë Using Functions¶

    def get_word_length(word: str) -> int:
    return len(word)

words = ["apple", "banana", "kiwi"]
words.sort(key=get_word_length)

    Œª Lambda Functions¶

    # Sort by length
words.sort(key=lambda word: len(word))

# Sort by last character
words.sort(key=lambda x: x[-1])

    üÜï Sorted Function¶

    üìù Basic Usage¶

    # Creates new sorted list
original = [3, 1, 4]
new_list = sorted(original)  # original unchanged

    üîß Advanced Usage¶

    # With reverse
sorted(numbers, reverse=True)

# With key function
sorted(numbers, key=abs)  # Sort by absolute value

    üí° Implementation Notes¶

    üîç Key Differences¶

    • .sort():
      • In-place modification
      • List method only
      • Returns None
    • sorted():
      • Creates new list
      • Works with any iterable
      • Returns sorted list

    ‚ö†Ô∏è Common Pitfalls¶

    # WRONG ‚ùå
sorted_list = list.sort()  # Returns None!

# CORRECT ‚úÖ
list.sort()
# OR
sorted_list = sorted(list)

    üöÄ Quick Reference¶

    # Basic sorting
list.sort()                    # In-place, ascending
sorted(list)                   # New list, ascending

# Reverse sorting
list.sort(reverse=True)        # In-place, descending
sorted(list, reverse=True)     # New list, descending

# Custom sorting
list.sort(key=len)            # By length
list.sort(key=lambda x: x[1])  # By second element

    üéØ Key Takeaway: Choose .sort() for in-place modification and sorted() when you need to preserve the original list or work with other iterables.
