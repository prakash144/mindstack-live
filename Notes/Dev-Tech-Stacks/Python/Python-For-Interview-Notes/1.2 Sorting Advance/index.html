    üîÑ Basic Sorting¶

    üìù .sort() Method¶

    • ‚ú® Key Features: python nums = [5, 3, 6, 2, 1] nums.sort() # [1, 2, 3, 5, 6]
      • Time Complexity: O(n log n)
      • Space Complexity: O(1)
      • Modifies original list
      • Returns None

    üî§ String Sorting¶

    # Interview Tip: Often used in anagram problems
words = ["eat", "tea", "tan", "ate", "nat", "bat"]
words.sort()  # ['ate', 'bat', 'eat', 'nat', 'tan', 'tea']

    üéØ Interview Patterns¶

    üìä Frequency Sort Pattern¶

    # Common in Top K Frequent Elements
from collections import Counter

def frequencySort(nums):
    # Count frequencies
    count = Counter(nums)
    # Sort by frequency, then by value
    return sorted(nums, key=lambda x: (-count[x], x))

    üîç Custom Interval Sort¶

    # Common in Merge Intervals problems
intervals = [[1,3], [2,6], [8,10]]
intervals.sort(key=lambda x: x[0])  # Sort by start time

    üìà Priority-based Sort¶

    # Common in Task Scheduler problems
tasks = ['A', 'A', 'B', 'B', 'C']
task_counts = Counter(tasks)
sorted_tasks = sorted(task_counts.items(), key=lambda x: (-x[1], x[0]))

    üéØ Custom Sorting¶

    üîë Common Interview Patterns¶

    # Sort matrix by row sum
matrix = [[1,2], [3,1], [2,5]]
matrix.sort(key=sum)

# Sort strings by custom order
order = "cba"
words = ["apple", "banana"]
words.sort(key=lambda x: [order.index(c) for c in x])

    Œª Advanced Lambda Patterns¶

    # Multiple criteria sort
people = [("John", 25), ("Jane", 25), ("Bob", 20)]
people.sort(key=lambda x: (x[1], x[0]))  # Sort by age, then name

# Custom comparison
nums = [-4, -1, 0, 3, 10]
nums.sort(key=lambda x: (abs(x), x))  # Sort by absolute value

    # Space-efficient approach

# Time-efficient approach (when need original)

    üöÄ Common Interview Tricks¶

    # Sort in reverse for efficiency
nums.sort(reverse=True)  # Better than sorting and reversing

# Custom comparison for strings
words.sort(key=str.lower)  # Case-insensitive sort

# Sort with multiple conditions
students.sort(key=lambda x: (-x[0], x[1]))  # Grade DESC, name ASC

    ‚ö†Ô∏è Interview Gotchas¶

    • Always check if input is None/empty
    • Consider stability requirements
    • Watch for duplicate elements
    • Remember sort() modifies in-place

    üéØ Quick Reference for Interviews¶

    # Common Patterns
list.sort(key=len)                 # Sort by length
list.sort(key=lambda x: -x)        # Reverse numeric sort
list.sort(key=lambda x: (x[0], x[1])) # Multi-key sort

# Interview Favorites
intervals.sort(key=lambda x: x[0])  # Interval problems
pairs.sort(key=lambda x: x[1])      # Greedy algorithms
strings.sort(key=str.lower)         # Case-insensitive

    üéØ Key Interview Notes: - Choose .sort() for in-place modification (O(1) space) - Use sorted() when you need to preserve the original list (O(n) space) - Always mention time complexity: O(n log n) - Consider using Counter for frequency-based sorting - Remember: sometimes linear solutions exist!
