    🚀 Debugging Production Issues¶

    📑 Table of Contents¶

    • Incident Management – Handling Outages
    • Root Cause Analysis (RCA)
    • Distributed Tracing – Debugging Microservices
    • Performance Optimization
    • 🔹 Key Takeaways

    1️⃣ Incident Management – Handling Outages 🛑¶

    ✅ Stay Calm & Follow SOPs – Follow a well-defined incident response plan.
    ✅ Triage & Prioritize – Identify if it's critical, major, or minor.
    ✅ Communicate Clearly – Keep stakeholders updated (internal & external).
    ✅ Use Monitoring & Logging – Check Prometheus, Grafana, ELK, or Splunk for anomalies.
    ✅ Mitigate First, Investigate Later – Apply temporary fixes to restore service quickly.
    ✅ Postmortem Reports – Document root cause, impact, and preventive measures.

    🛠 Tools & Commands for Incident Management¶

    journalctl -u service_name --since "1 hour ago"  # Check service logs
kubectl get pods --namespace=prod  # Check pod status in Kubernetes
dmesg -T | tail -20  # Check recent kernel logs for crashes

    2️⃣ Root Cause Analysis (RCA) 🔍¶

    ✅ Reproduce the Issue – Try to replicate in staging before touching production.
    ✅ Analyze Logs & Metrics – Identify error patterns, latencies, and spikes.
    ✅ Check Deployment Changes – Was there a recent push or config change?
    ✅ Use Debugging Tools – GDB, strace, lsof, perf for deep system insights.
    ✅ Run Profiling & Tracing – Identify CPU, memory, and I/O bottlenecks.

    🛠 Tools for Root Cause Analysis¶

    strace -p <pid>  # Trace system calls of a process
lsof -i :8080  # Find processes using port 8080
top / htop  # Check CPU and memory usage

    3️⃣ Distributed Tracing – Debugging Microservices 🕵️¶

    ✅ Why? Microservices introduce latency, dependency failures, and bottlenecks.

    ✅ Tools: - Jaeger 🏹 – Open-source end-to-end distributed tracing - Zipkin 🔗 – Lightweight tracing for microservices

    ✅ How? - Instrument services to send trace IDs with requests. - Analyze spans to find slow or failing components. - Identify network latency or database slowdowns.

    🛠 Commands for Tracing¶

    kubectl port-forward service/jaeger-query 16686:16686  # Access Jaeger UI
kubectl logs pod-name --tail=50  # Fetch last 50 logs of a pod

    4️⃣ Performance Optimization 🚀¶

    ✅ Memory Profiling - Identify memory leaks & inefficient allocations. - Use Valgrind, Heaptrack, or Python’s memory_profiler. - Reduce GC pressure in high-throughput applications.

    ✅ CPU Analysis - Use perf, eBPF, or flame graphs to profile CPU-intensive tasks. - Optimize hot paths and reduce unnecessary computation. - Enable async & parallel processing where applicable.

    ✅ Database Query Optimizations - Index frequently queried columns for faster lookups. - Avoid N+1 query issues using batching. - Use connection pooling for better DB performance. - Monitor slow queries with:

    EXPLAIN ANALYZE SELECT * FROM users WHERE email='user@example.com';

    🛠 Performance Debugging Tools¶

    top / htop  # Identify high CPU/memory-consuming processes
perf record -F 99 -p <pid> -g -- sleep 10  # Capture performance profile
iostat -xz 1  # Monitor disk I/O stats
EXPLAIN ANALYZE  # Analyze SQL query performance

    🔹 Key Takeaways¶

    ✅ Proactive monitoring reduces firefighting – Set up alerts for anomalies.
    ✅ Use distributed tracing in microservices – Helps in identifying slow requests.
    ✅ Always have a rollback plan – Quick fixes should not make things worse.
    ✅ Run postmortems after every major incident – Learn & improve system resilience.

    🎯 Stay prepared, monitor effectively, and keep production stable! 🚀
