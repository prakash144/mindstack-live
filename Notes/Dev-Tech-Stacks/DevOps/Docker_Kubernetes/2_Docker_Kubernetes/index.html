    üîπ Docker Basics¶

    ‚úÖ Key Concepts¶

    • Image üì¶: A lightweight, standalone package with everything needed to run software.
    • Container üöÄ: A running instance of a Docker image.
    • Dockerfile üìù: A script to build Docker images.
    • Docker Compose üìú: A tool to define and run multi-container applications.

    üî• Essential Commands¶

    Installation & Setup¶

    docker --version   # Check Docker version

    Images Management¶

    docker images      # List images
docker pull <image>  # Pull image
docker rmi <image>   # Remove image

    Container Operations¶

    docker ps        # List running containers
docker ps -a     # List all containers
docker run -d --name my_container my_image  # Run a container
docker stop my_container   # Stop a container
docker start my_container  # Start a container
docker rm my_container     # Remove a container
docker logs my_container   # View logs

    üîπ Intermediate Docker¶

    üèóÔ∏è Networking & Volumes¶

    docker network ls   # List networks

docker volume ls   # List volumes
docker volume create my_volume   # Create a volume
docker run -v my_volume:/data my_image   # Mount volume

    üìå Docker Compose¶

    sudo curl -L "https://github.com/docker/compose/releases/download/<version>/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose
sudo chmod +x /usr/local/bin/docker-compose
docker-compose --version   # Check Docker Compose version
docker-compose up -d   # Run Compose
docker-compose down    # Stop Compose

    üîπ Kubernetes Basics¶

    ‚úÖ Key Concepts¶

    • Pod üè†: The smallest deployable unit.
    • Node üíª: Machine running Kubernetes workloads.
    • Cluster üåê: A set of nodes managed by Kubernetes.
    • Deployment üì¶: Ensures a specified number of pods run.
    • Service üåâ: Exposes a set of pods as a network service.
    • ConfigMap & Secret üîë: Store configuration & sensitive data.

    üî• Essential Commands¶

    kubectl cluster-info  # Check cluster status
kubectl get nodes  # List nodes
kubectl create namespace my_namespace  # Create namespace
kubectl get namespaces  # List namespaces

    üîπ Intermediate Kubernetes¶

    üìå Working with Pods¶

    kubectl get pods   # List all pods
kubectl describe pod my_pod   # Describe a pod
kubectl delete pod my_pod   # Delete a pod

    üìå Deployments¶

    kubectl create deployment my_app --image=my_image  # Create deployment
kubectl scale deployment my_app --replicas=3  # Scale deployment
kubectl rollout undo deployment my_app  # Rollback deployment

    üìå Services & Networking¶

    kubectl expose deployment my_app --type=LoadBalancer --port=80  # Expose as service
kubectl get services   # List services

    üîπ Expert Kubernetes¶

    üî• Helm (Kubernetes Package Manager)¶

    curl https://raw.githubusercontent.com/helm/helm/master/scripts/get-helm-3 | bash  # Install Helm

    üî• Role-Based Access Control (RBAC)¶

    kubectl get roles -n my_namespace  # List roles
kubectl create rolebinding my_binding --role=my_role --user=my_user -n my_namespace  # Create binding

    üî• Monitoring & Debugging¶

    kubectl logs my_pod  # View logs
kubectl exec -it my_pod -- /bin/sh  # Execute command in pod
kubectl get events --sort-by=.metadata.creationTimestamp  # Get events
