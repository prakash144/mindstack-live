    🐳 Docker Mastery: Beginner to Expert¶

    🔹 Docker Basics¶

    ✅ Key Concepts¶

    • Image 📦: A lightweight, standalone package with everything needed to run software.
    • Container 🚀: A running instance of a Docker image.
    • Dockerfile 📝: A script to build Docker images.
    • Docker Compose 📜: A tool to define and run multi-container applications.

    🔥 Essential Commands¶

    Installation & Setup¶

    docker --version   # Check Docker version

    Images Management¶

    docker images      # List images
docker pull <image>  # Pull image
docker rmi <image>   # Remove image

    Container Operations¶

    docker ps        # List running containers
docker ps -a     # List all containers
docker run -d --name my_container my_image  # Run a container
docker stop my_container   # Stop a container
docker start my_container  # Start a container
docker rm my_container     # Remove a container
docker logs my_container   # View logs

    🔹 Intermediate Docker¶

    🏗️ Networking & Volumes¶

    docker network ls   # List networks
docker network create my_network   # Create a network
docker volume ls   # List volumes
docker volume create my_volume   # Create a volume
docker run -v my_volume:/data my_image   # Mount volume

    📌 Docker Compose¶

    sudo curl -L "https://github.com/docker/compose/releases/download/<version>/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose
sudo chmod +x /usr/local/bin/docker-compose
docker-compose --version   # Check Docker Compose version
docker-compose up -d   # Run Compose
docker-compose down    # Stop Compose

    🔹 Advanced Docker¶

    ⚙️ Docker Swarm & Security¶

    docker swarm init  # Initialize Swarm mode
docker node ls  # List Swarm nodes
docker stack deploy -c docker-compose.yml my_stack  # Deploy stack
docker secret create my_secret -  # Create a secret
docker secret ls   # List secrets

    🛡️ Best Practices¶

    ✅ Store Dockerfiles in Git 📂
    ✅ Keep images lightweight 📏
    ✅ Scan images for vulnerabilities 🔍
    ✅ Automate with CI/CD 🚀
    ✅ Monitor & Log 🖥️

    ☸️ Kubernetes Mastery: Beginner to Expert¶

    🔹 Kubernetes Basics¶

    ✅ Key Concepts¶

    • Pod 🏠: The smallest deployable unit.
    • Node 💻: Machine running Kubernetes workloads.
    • Cluster 🌐: A set of nodes managed by Kubernetes.
    • Deployment 📦: Ensures a specified number of pods run.
    • Service 🌉: Exposes a set of pods as a network service.
    • ConfigMap & Secret 🔑: Store configuration & sensitive data.

    🔥 Essential Commands¶

    kubectl cluster-info  # Check cluster status
kubectl get nodes  # List nodes
kubectl create namespace my_namespace  # Create namespace
kubectl get namespaces  # List namespaces

    🔹 Intermediate Kubernetes¶

    📌 Working with Pods¶

    kubectl get pods   # List all pods
kubectl describe pod my_pod   # Describe a pod
kubectl delete pod my_pod   # Delete a pod

    📌 Deployments¶

    kubectl create deployment my_app --image=my_image  # Create deployment
kubectl scale deployment my_app --replicas=3  # Scale deployment
kubectl rollout undo deployment my_app  # Rollback deployment

    📌 Services & Networking¶

    kubectl expose deployment my_app --type=LoadBalancer --port=80  # Expose as service
kubectl get services   # List services

    🔹 Expert Kubernetes¶

    🔥 Helm (Kubernetes Package Manager)¶

    curl https://raw.githubusercontent.com/helm/helm/master/scripts/get-helm-3 | bash  # Install Helm
helm repo add stable https://charts.helm.sh/stable  # Add Helm repo
helm install my_app stable/chart_name  # Install an app
helm list  # List installed releases

    🔥 Role-Based Access Control (RBAC)¶

    kubectl get roles -n my_namespace  # List roles
kubectl create rolebinding my_binding --role=my_role --user=my_user -n my_namespace  # Create binding

    🔥 Monitoring & Debugging¶

    kubectl logs my_pod  # View logs
kubectl exec -it my_pod -- /bin/sh  # Execute command in pod
kubectl get events --sort-by=.metadata.creationTimestamp  # Get events

    ☁️ AWS & GCP Considerations¶

    ✅ AWS EKS (Elastic Kubernetes Service)¶

    • Use IAM Roles for pod security.
    • Configure VPC and Security Groups for access control.
    • Use Auto Scaling Groups for efficient resource allocation.
    • Monitor with CloudWatch & Prometheus.

    ✅ GCP GKE (Google Kubernetes Engine)¶

    • Use Google IAM & Workload Identity.
    • Configure VPC-native clusters for security.
    • Enable Autoscaling and Cloud Logging.
