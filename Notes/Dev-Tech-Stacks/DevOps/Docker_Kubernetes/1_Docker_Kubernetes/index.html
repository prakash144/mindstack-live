    1Ô∏è‚É£ What is Docker? üê≥¶

    • Docker is a containerization platform that packages applications and dependencies into a single unit (container).
    • Ensures consistency across different environments (dev, test, prod).
    • Lightweight, fast, and isolates apps from OS dependencies.
    • Supports Microservices architecture by enabling easy scaling & deployment.

    üèóÔ∏è Key Docker Concepts¶

    • Image üì¶: Blueprint of a container (built from Dockerfile).
    • Container üè†: Running instance of an image.
    • Dockerfile üìú: Script to automate container creation.
    • Registry üèõÔ∏è: Storage for images (Docker Hub, AWS ECR, GCP Container Registry).
    • Volumes üóÑÔ∏è: Persistent storage for containers.

    üõ†Ô∏è Must-Know Docker Commands¶

    # Install Docker
sudo apt install docker.io  # Ubuntu
brew install --cask docker   # MacOS

# Check Docker version
docker --version

# Run a container
docker run -d -p 8080:80 nginx

# List running containers
docker ps

# Stop and remove a container
docker stop <container_id>
docker rm <container_id>

# Build an image
docker build -t my-app .

# Push an image to Docker Hub
docker tag my-app:latest mydockerhubuser/my-app:latest
docker push mydockerhubuser/my-app:latest

    2Ô∏è‚É£ What is Kubernetes? ‚ò∏Ô∏è¶

    • Kubernetes (K8s) is an open-source container orchestration tool for deploying, scaling, and managing containers.
    • Automates container deployment, load balancing, scaling, and self-healing.
    • Works across cloud providers (AWS, GCP, Azure) and on-prem environments.

    üèóÔ∏è Key Kubernetes Concepts¶

    • Pod üè†: Smallest deployable unit, contains one or more containers.
    • Node üñ•Ô∏è: A single machine in a Kubernetes cluster (Master & Worker Nodes).
    • Cluster üèóÔ∏è: A group of nodes managing containers.
    • Service üåç: Exposes an application inside/outside the cluster.
    • Ingress üåê: Manages external access to services.

    üõ†Ô∏è Must-Know Kubernetes Commands¶

    # Get cluster info
kubectl cluster-info

# Get nodes
kubectl get nodes

# Create a pod
kubectl run my-app --image=nginx --port=80

# List running pods
kubectl get pods

# Deploy an application
kubectl create deployment my-app --image=nginx

# Scale deployment
kubectl scale deployment my-app --replicas=3
