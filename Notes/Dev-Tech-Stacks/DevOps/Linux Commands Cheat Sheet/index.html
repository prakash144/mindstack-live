    🚀 Linux Commands Cheat Sheet¶

    A structured cheat sheet categorized into Beginner, Advanced, and Expert levels. It includes essential commands for development, debugging, and production environments. 💻

    • Advanced Commands

    🟢 Beginner Commands (Basic Usage)¶

    Command Description
    ls 📂 List files and directories
    pwd 📍 Print current working directory
    cd 🔄 Change directory
    mkdir <dir> 📁 Create a new directory
    rmdir <dir> ❌ Remove an empty directory
    rm <file> 🗑 Delete a file
    rm -r <dir> 🚨 Delete a directory (with files)
    cp <src> <dest> 📄 Copy file/directory
    mv <src> <dest> 📂 Move/Rename file/directory
    cat <file> 📜 View file content
    nano <file> ✍ Open file in nano editor
    vi <file> 📝 Open file in vi editor
    echo "text" 📢 Print text to terminal
    touch <file> 🆕 Create a new empty file
    find . -name <file> 🔍 Search for a file in current directory
    grep "text" <file> 🔎 Search for a pattern inside a file
    history 📜 Show command history
    clear 🧹 Clear the terminal screen

    🔵 Advanced Commands (System & File Management)¶

    Command Description
    df -h 💾 Check disk space usage
    du -sh <dir> 📊 Show directory size
    free -m 🧠 Show memory usage
    ps aux 🔍 View running processes
    top / htop 🚀 Monitor system resources
    kill <PID> ❌ Terminate a process by ID
    pkill <name> 🔫 Kill process by name
    chmod 755 <file> 🔐 Change file permissions
    chown user:group <file> 👥 Change file ownership
    ln -s <src> <dest> 🔗 Create symbolic link
    tar -cvf <archive.tar> <dir> 📦 Create tar archive
    tar -xvf <archive.tar> 📂 Extract tar archive
    zip -r <file.zip> <dir> 🗜 Compress files (ZIP)
    unzip <file.zip> 📂 Extract ZIP file
    scp <file> user@host:/path 🔄 Securely copy file to remote
    rsync -av <src> <dest> 🔄 Sync files/directories
    mount /dev/sdb1 /mnt 🔗 Mount a drive
    umount /mnt 🔗 Unmount a drive
    crontab -e ⏳ Edit scheduled tasks (cron jobs)

    🔴 Expert Commands (Networking, Debugging & Production)¶

    📡 Networking & System Performance¶

    Command Description
    ifconfig / ip a 🌐 Show network interfaces
    ping <host> 🔄 Check connectivity
    netstat -tulnp 🔍 View active network connections
    ss -tulnp 📡 Show listening ports (better than netstat)
    curl -I <URL> 🔍 Fetch HTTP headers
    wget <URL> 📥 Download file from web
    nmap -sP <IP-range> 🕵 Scan network for live hosts
    traceroute <host> 🌍 Show route to destination
    dig <domain> 🔍 Get DNS information
    nslookup <domain> 🌐 Get domain IP address
    tcpdump -i eth0 📡 Capture network traffic

    🛠 Debugging & Logs¶

    Command Description
    journalctl -xe 📜 View system logs
    tail -f /var/log/syslog 📝 Monitor logs in real-time
    strace -p <PID> 🕵 Trace system calls of a process
    lsof -i :80 🔎 List open ports (example: port 80)
    watch -n 2 free -m 👀 Monitor free memory every 2 sec
    iotop 📊 Show disk I/O usage by processes
    iostat -x 1 ⚡ Monitor CPU & disk I/O

    🔥 Development & Production Commands¶

    🛠 Development¶

    Command Description
    git clone <repo> 🛠 Clone a Git repository
    git pull 🔄 Fetch latest code
    git commit -m "message" 📜 Commit changes
    git push origin main 🚀 Push changes
    python3 -m http.server 🌍 Start local web server
    node server.js 🟢 Start a Node.js server
    docker ps 🐳 List running Docker containers
    docker logs <container> 📝 View container logs

    ⚙️ Production Debugging & Performance¶

    Command Description
    uptime ⏳ Show system uptime
    vmstat 1 📊 Monitor CPU & memory stats
    sar -u 2 5 📊 Get CPU usage history
    who 👤 Show logged-in users
    ps -eo pid,ppid,cmd,%mem,%cpu --sort=-%cpu 🔍 Find high CPU processes
    systemctl restart <service> 🔄 Restart a service
    systemctl status <service> ✅ Check service status
    fail2ban-client status 🛡 Check banned IPs

    📝 Best Practices¶

    ✅ Use man <command> – Get detailed info about a command
    ✅ Use && for chaining – cd /var/www && ls
    ✅ Use nohup or screen – Keep commands running in the background
    ✅ Use alias for shortcuts – alias ll='ls -la'
    ✅ Monitor logs & processes regularly

    🚀 Final Thoughts¶

    This cheat sheet covers everything from basic Linux commands to advanced debugging and production monitoring. Mastering these will prepare you for roles like DevOps Engineer, Cloud Architect, and Linux Administrator.
