    🔥 Security Principles – Quick Revision Notes¶

    ✅ 1. CIA Triad (Core Security Principles)¶

    • 🔒 Confidentiality: Ensures that data is accessed only by authorized individuals.
      • 🔹 Techniques: Encryption, Access Controls, MFA.
    • 🛡️ Integrity: Ensures data is accurate and unaltered.
      • 🔹 Techniques: Hashing, Digital Signatures, Checksums.
    • 🔓 Availability: Ensures systems and data are accessible when needed.
      • 🔹 Techniques: Redundancy, Failover, DDoS protection.

    🔥 2. Authentication & Authorization¶

    • 🔑 Authentication: Verifies the identity of a user or system.
      • 🔹 Types:
        • 🔐 Password-based
        • 📲 Multi-Factor Authentication (MFA)
        • 🔏 Biometric (Fingerprint, Face ID)
    • ✅ Authorization: Determines what actions a user is allowed to perform.
      • 🔹 RBAC: Role-Based Access Control.
      • 🔹 ABAC: Attribute-Based Access Control.

    ⚙️ 3. Security Models¶

    • 📊 Bell-LaPadula Model: Enforces confidentiality with "no read-up, no write-down".
    • 🔐 Biba Model: Enforces integrity with "no write-up, no read-down".
    • ⚖️ Clark-Wilson Model: Enforces both integrity and access control by separation of duties.
    • 🚫 Zero Trust Model: Assumes no implicit trust, continuous verification.

    🛡️ 4. Encryption & Cryptography¶

    • 🔹 Symmetric Encryption:
      • 🔐 Single key for encryption and decryption.
      • 🔥 Algorithms: AES, DES, 3DES.
    • 🔹 Asymmetric Encryption:
      • 🔑 Public and private key pair.
      • 🔥 Algorithms: RSA, ECC.
    • 🔹 Hashing:
      • 🛡️ One-way function, ensures integrity.
      • 🔥 Algorithms: SHA-256, MD5 (deprecated).
    • 🔹 Digital Signatures:
      • ✅ Verifies authenticity and integrity.
      • 🔥 RSA, DSA algorithms.

    🚦 5. Secure Coding Practices¶

    • ✅ Input Validation: Prevents SQL Injection, XSS.
    • 🔥 Output Encoding: Mitigates XSS attacks.
    • 🔒 Least Privilege: Grant minimal access necessary.
    • 🚫 Avoid Hardcoding Secrets: Use secure vaults.
    • 🔐 Secure APIs:
      • ✅ Rate limiting and throttling.
      • 🔥 OAuth and token-based authentication.

    🚀 6. Secure Development Lifecycle (SDLC)¶

    • 🔹 Requirements Gathering: Define security needs upfront.
    • 🔹 Design: Incorporate security principles.
    • 🔹 Implementation: Secure coding practices.
    • 🔹 Testing: Penetration testing, vulnerability scanning.
    • 🔹 Deployment: Secure deployment and monitoring.
    • 🔹 Maintenance: Regular patching and updates.

    🔥 7. Network Security¶

    • 🔹 Firewalls: Blocks unauthorized traffic.
    • 🔹 IDS/IPS: Detects and prevents suspicious activities.
    • 🔹 VPN: Encrypts network traffic.
    • 🔹 Network Segmentation: Reduces attack surface.
    • 🔹 DDoS Protection: Prevents service disruptions.

    ⚙️ 8. Cloud Security Best Practices¶

    • 🔒 Encryption: At rest and in transit.
    • ✅ IAM (Identity and Access Management): Restricts access.
    • 🌐 VPC (Virtual Private Cloud): Isolates resources.
    • 🔥 Data Loss Prevention (DLP): Prevents data leaks.
    • 🚀 Monitoring & Logging: Detects anomalies.

    🔥 9. Security Testing Techniques¶

    • 🔥 Penetration Testing: Simulated attacks to find vulnerabilities.
    • 🔹 Static Analysis: Reviews code without execution.
    • 🔹 Dynamic Analysis: Reviews running application behavior.
    • 🔥 Fuzz Testing: Sends random data to find bugs.

    ✅ 10. Incident Response (IR)¶

    • 🛡️ Preparation: Policies, playbooks, and tools.
    • 🚦 Detection and Analysis: Identify incidents.
    • 🔥 Containment: Isolate affected systems.
    • 🔓 Eradication: Remove threats.
    • 🔁 Recovery: Restore services.
    • ✅ Post-Incident Review: Learn and improve.
