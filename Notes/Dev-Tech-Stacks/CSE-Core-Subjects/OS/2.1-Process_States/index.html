    📌 Process and Its States¶

    📜 Table of Contents¶

    🔹 Introduction¶

    • A process is a program in execution, utilizing system resources like CPU, memory, and I/O devices.
    • Each process has a Process Control Block (PCB) storing its execution details.
    • 🏠 Real-Life Analogy: A recipe 📝 (program) becomes a dish being cooked 🍳 (process) when a chef (CPU) executes it.

    🔹 Process in Memory¶

    When a user runs a program, it is loaded into memory as a process with: - 📄 Text Segment: Executable code. - 📊 Data Segment: Global & static variables. - 📦 Heap: Dynamic memory allocation. - 🗂️ Stack: Local variables, function calls, return addresses.

    🔹 Creation of a Process¶

    🛠️ Steps when a program is launched: 1. OS loads the program into main memory. 2. OS assigns a unique Process ID (PID). 3. A Process Control Block (PCB) is created, storing: - Process state - Registers & program counter - Allocated resources

    🔹 Process States¶

    A process transitions between various states during its lifecycle: - 🆕 New: Being created; not yet ready to run. - ✅ Ready: Loaded into memory, waiting for CPU. - ⚡ Running: Currently being executed by the CPU. - ⏳ Waiting (Blocked): Waiting for I/O or resource availability. - 🛑 Suspended: Temporarily paused/swapped to disk. - ❌ Terminated: Completed execution, resources released.

    🔹 State Transitions¶

    Transition Description
    New → Ready Process creation completed, ready to run.
    Ready → Running CPU assigned by the scheduler.
    Running → Waiting Waiting for I/O or resource.
    Running → Ready Preempted by scheduler (e.g., time slice ends).
    Waiting → Ready I/O completed, resource available.
    Running → Terminated Process execution completed.
    Ready/Waiting → Suspended Process paused/swapped to disk.
    Suspended → Ready/Waiting Process brought back into memory.

    🔹 Advantages of Process Management¶

    ✅ Efficient CPU Utilization: Balances workload using scheduling. ✅ Multitasking & Resource Sharing: Enables smooth parallel execution. ✅ Improved System Responsiveness: Manages waiting processes efficiently.

    📌 Example: - 🎥 Video Player Application - Running: Decoding video. - Waiting: Fetching file from disk. - Ready: Resuming playback after loading.

    🔹 Process Control Block (PCB)¶

    🗂️ The PCB is a data structure storing all necessary process details: - 📍 Process State: Ready, running, waiting, etc. - 🔢 Process ID (PID): Unique identifier. - 📌 Program Counter (PC): Next instruction address. - ⚙️ CPU Registers: Accumulators, stack pointers, condition codes. - 🗃️ Memory Management Info: Base/limit registers, page tables. - 📊 Scheduling Info: CPU usage, priority, wait times. - 📁 I/O Status: Open files, devices in use.

    🔹 Advanced Perspective¶

    🔹 Modern OS Features: - Multi-core Scheduling: Each core accesses process PCBs. - Priority-based Scheduling: Processes assigned priority levels. - Load Balancing: Distributes workloads efficiently. - Security & Access Control: Process permissions managed via PCB.

    🔹 Conclusion¶

    • ✅ The Process Control Block (PCB) is the backbone of process management.
    • ✅ Process states & transitions enable efficient multitasking & CPU usage.
    • ✅ Advanced OS techniques leverage PCBs for better scheduling, security, and performance.

    📌 Key Takeaway: Understanding processes and their states is critical for mastering OS concepts and designing efficient computing systems! 🚀
