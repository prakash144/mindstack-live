    üîπ System Calls Overview¶

    System Calls serve as an interface between user applications and the operating system kernel, enabling secure access to hardware and resources.

    üìû System Call as a Customer Service Representative: - A user program (customer) requests a service (e.g., open a file, allocate memory). - The OS (company) processes the request via a system call (service rep). - The service rep returns the result safely (grants or denies access).

    üõ† User Mode ‚Üí Kernel Mode Transition: 1. User Mode Execution: Applications operate in a restricted environment. 2. System Call Invocation: A request is made (e.g., open() file). 3. Mode Switch: Control shifts to kernel mode. 4. Request Processing: The OS validates, executes, and manages resources. 5. Result Delivery: Control returns to user mode with a response.
