    üè† 1. Network Address Translation (NAT)¶

    Purpose: Enables multiple devices in a private network to share a single public IP address, optimizing IPv4 usage and enhancing security.

    üîÑ Types of NAT¶

    • üîπ Static NAT: One-to-one mapping between private and public IPs (e.g., web servers).
    • üîπ Dynamic NAT: Maps private IPs to a pool of public IPs dynamically.
    • üîπ Port Address Translation (PAT) (NAT Overload): Maps multiple private IPs to a single public IP using port numbers (common in homes and small offices).

    ‚úÖ Advantages¶

    ‚úî Conserves Public IPs ‚Üí Reduces IPv4 exhaustion.
    ‚úî Enhances Security ‚Üí Hides internal network structure.
    ‚úî Facilitates Internet Access ‚Üí Enables private networks to communicate with public networks.

    ‚ùå Limitations¶

    ‚ùå Latency Overhead ‚Üí Address translation introduces delays.
    ‚ùå Breaks End-to-End Connectivity ‚Üí Some protocols (VoIP, P2P, VPN) struggle with NAT.
    ‚ùå Limits Some Applications ‚Üí NAT traversal is needed for direct communication.
