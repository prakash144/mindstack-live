    📌 IP Addressing (IPv4 & IPv6)¶

    📚 Table of Contents¶

    📖 Overview¶

    • 🌐 Unique identifier for devices in a network.
    • 🚀 Ensures efficient data transmission & routing.
    • 📡 Functions at Network Layer of OSI & TCP/IP models.

    🎯 Purpose¶

    • 🆔 Identifies devices uniquely.
    • 📦 Routes data packets efficiently.
    • 🔗 Differentiates devices across LANs, WANs, & the Internet.

    🏛️ IPv4 Addressing¶

    📌 Structure¶

    • 🔢 32-bit address, dotted decimal notation (e.g., 192.168.1.1).
    • 📊 Composed of 4 octets (8 bits each), range 0-255 per octet.

    🏗️ IPv4 Header Components¶

    • 🏷️ Version: Identifies IP version (4).
    • 🏁 Header Length: Marks start of payload.
    • 🎯 Type of Service (ToS): Defines priority & QoS.
    • 📏 Total Length: Max 65,535 bytes.
    • 🔄 Fragmentation Fields: Identification, Flags, Fragment Offset.
    • ⏳ TTL: Prevents infinite looping.
    • 🛠️ Protocol: Identifies transport-layer protocol (TCP/6, UDP/17).
    • 🔍 Header Checksum: Ensures integrity.
    • 📌 Source & Destination Address: Identifies sender & receiver.
    • ⚙️ Options (Optional): Adds routing preferences.

    📊 Address Classes¶

    • 🅰️ Class A: Large networks (1.0.0.0 – 126.255.255.255).
    • 🅱️ Class B: Medium networks (128.0.0.0 – 191.255.255.255).
    • 🅲️ Class C: Small networks (192.0.0.0 – 223.255.255.255).
    • 🅳️ Class D: Multicast (224.0.0.0 – 239.255.255.255).
    • 🅴️ Class E: Experimental (240.0.0.0 – 255.255.255.255).

    ❌ Limitations¶

    • 📉 Address Exhaustion: Limited to ~4.3 billion addresses.
    • 🔓 Security Risks: No built-in encryption.
    • 📡 Scalability Issues: Requires NAT & subnetting.

    🏛️ IPv6 Addressing¶

    📌 Structure¶

    • 🔢 128-bit address, colon-hexadecimal notation (e.g., 2001:db8::1).
    • 🧩 Eight 16-bit segments, separated by colons.
    • ⚡ Supports shorthand notation for zero segments.

    🏗️ IPv6 Header Components¶

    • 🏷️ Version: Identifies IP version (6).
    • 🚦 Traffic Class: Defines priority & QoS.
    • 🔄 Flow Label: Optimizes routing.
    • 📦 Payload Length: Size of transmitted data.
    • 🛠️ Next Header: Identifies next protocol.
    • ⏳ Hop Limit: Prevents infinite looping (TTL equivalent).
    • 📌 Source & Destination Address: Identifies sender & receiver.

    ⭐ Features¶

    • 📈 Vast Address Space: ~340 undecillion addresses.
    • 🔒 Built-in Security: Supports IPsec.
    • 🤖 Auto-Configuration: Devices generate their own addresses.
    • 🚀 Efficient Routing: Reduces routing table sizes.

    🔄 Compatibility¶

    • ⚡ Dual-stack (IPv4 + IPv6) & Tunneling for transition.

    🔄 IPv4 vs IPv6 Comparison¶

    Feature IPv4 IPv6
    Address Length 32-bit 128-bit
    Notation Dotted decimal (192.168.1.1) Colon-hexadecimal (2001:db8::1)
    Address Space ~4.3 billion ~340 undecillion
    Security No built-in encryption Supports IPsec
    Header Size 20 bytes 40 bytes
    Fragmentation Done by routers Done by source device
    Routing Efficiency Less efficient More efficient

    🏗️ IP Addressing Techniques¶

    🛠 Subnetting¶

    • 🏷️ Definition: Splitting a large network into smaller subnets.
    • 🎯 Purpose:
      • 🚦 Reduces network congestion.
      • 🔐 Enhances security.
      • 📊 Optimizes address allocation.
    • 🔢 Method:
      • Uses subnet masks (e.g., 255.255.255.0 or /24 in CIDR notation).
      • 🏗 Example:
        • Original: 192.168.1.0/24 (254 usable IPs).
        • Subnets: 192.168.1.0/26, 192.168.1.64/26, etc.

    📈 Supernetting¶

    • 🏷️ Definition: Merging smaller networks into a larger one.
    • 🎯 Purpose:
      • 📉 Reduces routing table size.
      • 🚀 Improves efficiency.
    • 🔢 Method:
      • Uses CIDR (e.g., 192.168.0.0/22 aggregates multiple /24 subnets).

    ⚡ Address Allocation Methods¶

    • 🏛️ Classful Addressing: Uses predefined network classes.
    • 🔗 Classless Addressing (CIDR): Allows flexible subnetting.

    🌍 Applications¶

    • 🏢 IPv4: Legacy systems, private networks, small-scale deployments.
    • 🚀 IPv6: IoT, large-scale networks, modern internet.

    🏁 Conclusion¶

    ✅ IP addressing enables device identification & communication. ✅ IPv4 remains widely used, but IPv6 is essential for future scalability. ✅ Subnetting & supernetting optimize address allocation & network performance.
