    🛑 Exception Handling in Java¶

    🚀 1. What is Exception Handling?¶

    • Exception Handling is a mechanism to handle runtime errors gracefully, ensuring the program doesn't crash unexpectedly.
    • Exception: An event that disrupts the normal flow of the program.
    • Objective: To catch errors, prevent abnormal termination, and provide meaningful messages.

    🔥 2. Exception Hierarchy in Java¶

    • Java exceptions are derived from the Throwable class.
    • Hierarchy:
                   Throwable
               /      \
         Error       Exception
                   /         \
          Checked    Unchecked
    • Error:
      • Irrecoverable issues (e.g., OutOfMemoryError, StackOverflowError).
      • Should not be handled using exception handling.
    • Exception:
      • Represents recoverable issues.
      • Two types:
        • Checked Exception: Caught at compile-time.
        • Unchecked Exception: Caught at runtime.

    🔥 3. Types of Exceptions¶

    ✅ Checked Exceptions (Compile-time)¶

    • Checked at compile-time, mandatory to handle or declare using throws.
    • Examples:
      • IOException
      • SQLException
      • FileNotFoundException

    ⚠️ Unchecked Exceptions (Runtime)¶

    • Occur during runtime, not checked by the compiler.
    • Examples:
      • NullPointerException
      • ArrayIndexOutOfBoundsException
      • ArithmeticException

    🛠️ 4. Exception Handling Keywords¶

    🔥 Keyword ✅ Purpose 💡 Example Usage
    try Defines a block of code to test for exceptions. try { int a = 10 / 0; }
    catch Handles the exception thrown by try. catch (Exception e) {}
    finally Always executed (optional). Used for cleanup. finally { System.out.println(); }
    throw Used to throw an exception explicitly. throw new NullPointerException();
    throws Declares exceptions that may occur. void myMethod() throws IOException

    🔥 5. Try-Catch-Finally Block¶

    • Syntax:
    try {
    // Code that may throw an exception
} catch (ExceptionType e) {
    // Handle exception
} finally {
    // Optional cleanup block
}
    • Flow:
      • try: Code execution starts here.
      • catch: Executes if an exception is thrown.
      • finally: Executes always, whether an exception occurs or not.

    🚀 6. Throw vs Throws¶

    ✅ throw ⚠️ throws
    Used to throw an exception explicitly. Declares exceptions in method signature.
    Used within a method. Used with method declaration.
    Throws a single exception. Can declare multiple exceptions.
    Example: throw new NullPointerException(); Example: void myMethod() throws IOException

    🔥 7. Custom Exceptions¶

    • You can create your own exception classes by extending Exception or RuntimeException.
    • Syntax:
    class CustomException extends Exception {
    public CustomException(String message) {
        super(message);
    }
}

public class Main {
    public static void main(String[] args) throws CustomException {
        throw new CustomException("Custom Exception Occurred");
    }
}
    • When to use:
      • When existing exceptions are insufficient.
      • To create meaningful, specific exceptions for business logic.

    🚀 8. Best Practices for Exception Handling¶

    • ✅ Use specific exceptions instead of generic Exception or Throwable.
    • 🔥 Use multiple catch blocks for handling different exceptions separately.
    • 💡 Don’t suppress exceptions by using empty catch blocks.
    • ⚠️ Log exceptions properly with relevant information (message, stack trace).
    • 🛑 Avoid using exceptions for flow control as it is costly in performance.
    • 🚀 Use finally blocks for cleanup operations (closing files, releasing resources).
    • 🔥 Create meaningful custom exceptions for better debugging and maintenance.

    ✅ Key Takeaways: - Exception handling makes your code more robust and prevents crashes. - Proper handling improves code reliability and maintainability. - Use custom exceptions to handle application-specific errors effectively.
