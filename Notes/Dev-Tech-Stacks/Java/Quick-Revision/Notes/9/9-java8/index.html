    🔥 Java 8 Key Features - Pro Level Revision Notes¶

    🧠 Overview¶

    Java 8 introduced powerful functional programming capabilities, modern API enhancements, and performance improvements. It transformed Java from an OOP-only language to a hybrid of OOP + Functional.

    🧩 1. Lambdas (→)¶

    • 🔹 Lambda Expressions simplify the implementation of functional interfaces.
    • 🔹 Syntax: (args) -> expression or (args) -> { statements }
    • 🔹 Replaces anonymous classes, especially for one-method interfaces.
    • 💡 Behind the scenes: Compiled to invokedynamic bytecode via LambdaMetafactory.
    // Traditional
Runnable r = new Runnable() {
    public void run() {
        System.out.println("Running");
    }
};

// Java 8 Lambda
Runnable r = () -> System.out.println("Running");

    🔍 Best Practices:¶

    • Prefer for event handling, collection iteration, and functional pipelines.
    • Combine with Streams and functional interfaces like Function, Predicate, Supplier, Consumer.

    🌊 2. Streams API¶

    • 🔹 Provides a declarative, lazy, and parallelizable way to process collections.
    • 🔹 Supports operations like filter, map, reduce, collect, etc.
    • 🔹 Encourages stateless, immutable operations.
    List<String> names = Arrays.asList("Alice", "Bob", "Charlie");
names.stream()
     .filter(name -> name.startsWith("A"))
     .map(String::toUpperCase)
     .forEach(System.out::println); // Outputs: ALICE

    🚀 Stream Types:¶

    • Sequential Stream: collection.stream()
    • Parallel Stream: collection.parallelStream()

    🔍 Stream Best Practices:¶

    • Avoid stateful lambda expressions.
    • Use parallel streams cautiously — assess thread-safety and data size.

    🧩 3. Default Methods¶

    • 🔹 Enables method implementation in interfaces.
    • 🔹 Solves the diamond problem via compiler rules.
    • 🔹 Promotes interface evolution without breaking existing implementations.
    interface Vehicle {
    default void start() {
        System.out.println("Starting vehicle");
    }
}

    🔍 Key Points:¶

    • Can override in implementing classes.
    • Can use super to invoke interface default.
    • Static methods in interfaces are also allowed (static void utilMethod()).

    ❓ 4. Optional¶

    • 🔹 Container object that may or may not contain a value.
    • 🔹 Eliminates null checks and reduces NullPointerExceptions.
    Optional<String> name = Optional.of("Java");
name.ifPresent(System.out::println);

    🚀 Common Methods:¶

    • of(), ofNullable(), isPresent(), get(), orElse(), orElseGet(), map(), flatMap()

    🔍 Best Practices:¶

    • Use to return values from APIs that may be absent.
    • Avoid using Optional.get() directly.
    • Use chaining (map/flatMap) for elegant transformations.

    🛠️ Expert Tips¶

    • 🧪 Combine Lambdas + Streams for pipelined processing.
    • 🧠 Use method references (ClassName::method) for clean syntax.
    • 🔒 Consider thread-safety with parallel streams.
    • 📦 java.util.function package is core to functional programming in Java 8.
    • 🔍 Collectors.groupingBy, partitioningBy, and custom collectors = powerful aggregation tools.

    📌 Summary Table¶

    Feature Purpose Key Benefit
    Lambda Functional interface impl. Less boilerplate, cleaner syntax
    Stream API Declarative collection processing Lazy, efficient, chainable ops
    Default Methods Interface evolution Backward compatibility
    Optional Nullable value container Avoids null checks & NPE

    ✅ With Java 8, you're writing more concise, testable, and parallel-ready code.
