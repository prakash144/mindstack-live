    🌱 Core Spring Concepts – Revision Notes¶

    🔄 Inversion of Control (IoC)¶

    📌 Definition¶

    • A design principle where the control of object creation and dependency management is transferred from the application code to the Spring container.
    • Promotes loose coupling and better testability.

    ⚙️ How it works in Spring¶

    • The Spring IoC Container is responsible for:
      • Instantiating beans
      • Configuring beans
      • Wiring dependencies

    🧠 Expert Insight¶

    • Think of IoC as Hollywood Principle: "Don’t call us, we’ll call you."
    • Useful in replacing new keyword-based instantiation with container-managed instantiation.
    ApplicationContext context = new ClassPathXmlApplicationContext("beans.xml");
MyService service = context.getBean(MyService.class);

    🧩 Dependency Injection (DI)¶

    📌 Definition¶

    • A design pattern where the dependencies of an object are injected externally rather than being created internally.
    • Implements IoC.

    💡 Types of DI in Spring¶

    • Constructor Injection (preferred for mandatory dependencies)
    • Setter Injection (flexible, supports optional dependencies)
    • Field Injection (discouraged in production – harder to test & less explicit)

    ⚙️ Spring Annotations for DI¶

    • @Autowired – automatically injects dependencies
    • @Qualifier – used with @Autowired to resolve ambiguity
    • @Inject, @Resource – JSR-330 & JSR-250 annotations (optional alternatives)

    ✅ Constructor Injection Example¶

    @Component
public class OrderService {
    private final PaymentService paymentService;

    @Autowired
    public OrderService(PaymentService paymentService) {
        this.paymentService = paymentService;
    }
}

    ⚠️ Best Practices¶

    • Use constructor injection for required dependencies
    • Avoid field injection for testability and immutability
    • Keep configuration centralized and explicit (e.g., Java config classes or profiles)

    🧠 Quick Summary Table¶

    Concept Purpose Spring Mechanism Best Practice Tip
    IoC Shift control to container BeanFactory, ApplicationContext Delegate lifecycle to Spring container
    DI Inject dependencies externally @Autowired, XML, Java config Prefer constructor injection for clarity

    🛠️ For Experienced Developers¶

    • Use Java-based configuration over XML for type safety and better refactoring.
    • Combine profiles with DI to inject environment-specific beans.
    • Integrate with Spring Boot's auto-configuration to reduce boilerplate.
    @Configuration
@Profile("dev")
public class DevConfig {
    @Bean
    public DataSource dataSource() {
        return new HikariDataSource(...);
    }
}

    🧘‍♂️ Embrace Spring's IoC container as your core backbone — everything else (AOP, Data, MVC) builds on top of it.
