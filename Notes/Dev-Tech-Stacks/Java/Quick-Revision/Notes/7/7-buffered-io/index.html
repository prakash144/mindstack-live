    üîÑ Buffered I/O Streams in Java¶

    üß† Overview¶

    • Buffered I/O Streams improve I/O performance by minimizing native I/O calls.
    • They use an internal buffer to read/write data in bulk rather than byte-by-byte or char-by-char.
    • Used in scenarios involving large data processing, file operations, or network communications.

    üì• BufferedReader¶

    ‚úÖ Purpose:¶

    • Efficiently reads characters, arrays, or lines from a character input stream.

    üìå Key Features:¶

    • Wraps around Reader classes (e.g., FileReader).
    • Reads text line-by-line with readLine().
    • Helps reduce disk access latency.

    üß™ Example:¶

    BufferedReader reader = new BufferedReader(new FileReader("data.txt"));
String line;
while ((line = reader.readLine()) != null) {
    System.out.println(line);
}
reader.close();

    üßµ Threading:¶

    • Not thread-safe by default.
    • Wrap with Collections.synchronizedXXX() if shared.

    üì§ BufferedWriter¶

    ‚úÖ Purpose:¶

    • Efficiently writes characters, arrays, or strings to a character output stream.

    üìå Key Features:¶

    • Wraps around Writer classes (e.g., FileWriter).
    • Supports newLine() to write system-specific line separators.
    • Reduces number of write system calls.

    üß™ Example:¶

    BufferedWriter writer = new BufferedWriter(new FileWriter("output.txt"));
writer.write("Hello, world!");
writer.newLine();
writer.write("Buffered I/O is efficient!");
writer.close();

    üîÑ Buffer Flush:¶

    • Always call flush() or close() to ensure buffer is written to disk.

    üß† Deep Dive for Experienced Engineers¶

    ‚öôÔ∏è Internal Working:¶

    • Maintains a char array buffer (default 8192 chars).
    • On read(), fills buffer once and serves from memory.
    • On write(), fills buffer and flushes once full or on command.

    üìä Performance Tips:¶

    • Use for batch processing, not for small, frequent reads/writes.
    • Consider BufferedReader + InputStreamReader when reading bytes as characters.
    • Use try-with-resources for better resource management:
    try (BufferedReader br = new BufferedReader(new FileReader("file.txt"))) {
    // use br
}

    ‚úÖ Summary¶

    • Use BufferedReader/Writer when working with text-based I/O.
    • Ensure to flush/close writers to persist data.
    • Wrap readers/writers to boost performance in I/O intensive apps.
