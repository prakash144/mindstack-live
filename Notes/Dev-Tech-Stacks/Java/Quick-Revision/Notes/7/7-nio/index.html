    🔥 NIO (New I/O) in Java¶

    📌 Overview¶

    • Introduced in Java 1.4, enhanced in Java 7 (NIO.2).
    • Designed for scalable, non-blocking, buffer-oriented I/O.
    • Allows direct interaction with the underlying OS for high-performance I/O operations.

    📁 Path (java.nio.file.Path)¶

    🔹 Represents a location in the file system in an OS-independent way.

    • Created via Paths.get() or FileSystems.getDefault().getPath()
    Path path = Paths.get("/home/user/file.txt");
    • Supports path manipulation:
      • getFileName(), getParent(), resolve(), relativize()
    • Immutable and thread-safe.

    ✅ Use Case: Elegant file system traversal and manipulation in a cross-platform way.

    📂 Files (java.nio.file.Files)¶

    🔹 Utility class for common file operations.

    • Methods are static and work with Path:
    Files.readAllLines(path);
Files.copy(src, dest, StandardCopyOption.REPLACE_EXISTING);
Files.exists(path);
    • Supports:
      • File creation, deletion, copying, moving
      • Reading and writing text/binary
      • Metadata inspection (e.g., size, attributes)

    ✅ Use Case: High-level operations on files without manually handling streams.

    🔗 Channels (java.nio.channels)¶

    🔹 Abstraction for reading/writing data to/from buffers.

    • Types: FileChannel, SocketChannel, DatagramChannel, ServerSocketChannel
    • Supports non-blocking I/O and selectors for multiplexing.
    • FileChannel can map files to memory (memory-mapped I/O):
    FileChannel channel = FileChannel.open(path, StandardOpenOption.READ);
    • Supports scattering (read into multiple buffers) and gathering (write from multiple buffers).

    ✅ Use Case: High-throughput I/O, network communication, large file processing.

    📦 ByteBuffer (java.nio.ByteBuffer)¶

    🔹 Core of NIO buffer-based architecture.

    • Types: ByteBuffer, CharBuffer, IntBuffer, etc.
    • Stores raw binary data.
    • Key methods:
      • put(), get()
      • flip(), rewind(), clear(), compact()
    ByteBuffer buffer = ByteBuffer.allocate(1024);
buffer.put(data);
buffer.flip();
channel.write(buffer);
    • Can be direct (off-heap, faster) or heap-based.

    ✅ Use Case: Efficient binary data manipulation and zero-copy operations.

    🧠 Advanced Notes (For Experienced Engineers)¶

    • Zero-Copy: Using FileChannel.transferTo() / transferFrom() to avoid copying data between kernel and user space.
    • Memory-Mapped Files: FileChannel.map() maps files directly into memory — useful for large datasets.
    • Selectors & Multiplexing: With Selector, multiple channels can be managed with a single thread.
    • Thread-Safety: Buffers are not thread-safe. Use synchronization or thread-local buffers if needed.

    📌 Summary Table¶

    Component Description Key Methods / Classes
    Path File system path abstraction Paths.get(), resolve(), relativize()
    Files Utility class for file operations Files.readAllLines(), copy(), exists()
    Channel Connection to I/O source/sink FileChannel, SocketChannel, transferTo()
    ByteBuffer Memory buffer to read/write binary data put(), get(), flip(), compact()

    ✅ Best Practice: Combine Channels + Buffers + Selectors for performant, scalable I/O in real-world applications like file servers, log processors, and high-load network apps.
