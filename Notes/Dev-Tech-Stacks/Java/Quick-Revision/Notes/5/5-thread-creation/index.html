    5 thread creation

    🔥 Thread Creation in Java¶

    🚀 1. Introduction to Multithreading¶

    • Multithreading allows multiple parts of a program (threads) to execute concurrently.
    • Improves application performance by utilizing CPU cores efficiently.
    • Each thread runs in its own stack memory, allowing parallel execution.

    🔥 2. Ways to Create Threads in Java¶

    • Java provides two main ways to create threads:
      1. Extending the Thread class
      2. Implementing the Runnable interface

    ✅ 3. Extending the Thread Class¶

    • In this approach, you create a class that extends Thread and overrides the run() method.
    • Use the start() method to begin execution of the thread.

    ⚙️ Syntax:¶

    class MyThread extends Thread {
    public void run() {
        System.out.println("Thread is running...");
    }

    public static void main(String[] args) {
        MyThread thread = new MyThread();
        thread.start();  // Starts the new thread
    }
}

    🔥 Key Points:¶

    • The run() method contains the code to be executed by the thread.
    • start() method invokes the thread and calls the run() method internally.
    • Thread scheduling is handled by the JVM.

    ✅ Advantages: - Simplicity in syntax for small-scale applications.

    ❌ Disadvantages: - Single inheritance restriction: Since Java does not support multiple inheritance, you cannot extend another class.

    ✅ 4. Implementing the Runnable Interface¶

    • The Runnable interface has a single abstract method run() that must be implemented.
    • Create an instance of the Thread class by passing a Runnable object as a parameter.

    ⚙️ Syntax:¶

    class MyRunnable implements Runnable {
    public void run() {
        System.out.println("Runnable thread is running...");
    }

    public static void main(String[] args) {
        MyRunnable runnable = new MyRunnable();
        Thread thread = new Thread(runnable);
        thread.start();  // Starts the new thread
    }
}

    🔥 Key Points:¶

    • Runnable interface provides a way to achieve multithreading without inheritance.
    • Thread class takes the Runnable instance as an argument.
    • start() method is used to begin the execution.

    ✅ Advantages: - No inheritance limitation: The class can extend another class while still implementing Runnable. - Suitable for large-scale applications.

    ❌ Disadvantages: - Slightly more complex syntax compared to extending Thread.

    🔥 5. Differences Between Thread and Runnable¶

    🚀 Aspect ✅ Extending Thread 🔥 Implementing Runnable
    Inheritance Inherits from Thread class Implements Runnable interface
    Multiple Inheritance ❌ Not possible ✅ Possible
    Memory Overhead Slightly higher (creates new thread) Lower (uses shared thread object)
    Flexibility Less flexible More flexible
    Preferred For Small projects Large, scalable applications

    🚀 6. Best Practices for Thread Creation¶

    • Use Runnable over Thread: More flexible and allows multiple inheritance.
    • Prefer ExecutorService and Callable over raw thread creation for better management.
    • Use Thread pooling for managing multiple threads efficiently.
    • Always handle exceptions inside run() to avoid unexpected termination.
    • Use synchronized blocks to prevent race conditions in shared resources.

    ✅ 7. Key Takeaways¶

    • Use Runnable for better flexibility and scalability.
    • Avoid creating too many threads manually – use ExecutorService.
    • Always handle exceptions properly to prevent crashes.
    • Use synchronization where necessary to prevent race conditions.
