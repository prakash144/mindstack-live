    🛠️ Collections Utility Class in Java¶

    🚀 1. Overview¶

    • The Collections class is part of java.util package.
    • It provides static utility methods for performing operations on collections, such as:
      • Sorting
      • Searching
      • Synchronizing
      • Thread-safe operations
    • Enhances the functionality of the Collection framework with efficient and convenient methods.

    🔥 2. Key Methods with Examples¶

    ✅ 1. Sorting Collections – Collections.sort()¶

    • Used to sort elements of List, ArrayList, LinkedList in ascending or custom order.
    • Uses Timsort (a hybrid of merge sort and insertion sort).
    import java.util.*;

public class SortExample {
    public static void main(String[] args) {
        List<String> names = Arrays.asList("John", "Alice", "Bob");
        Collections.sort(names);   // Ascending order
        System.out.println(names); // Output: [Alice, Bob, John]
    }
}

    ✅ Custom Sorting with Comparator:

    Collections.sort(list, (a, b) -> b.compareTo(a));  // Descending order

    ✅ Sorting Primitive Arrays:

    int[] nums = {3, 1, 4, 1, 5};
Arrays.sort(nums);  // Output: [1, 1, 3, 4, 5]

    🔥 2. Binary Search – Collections.binarySearch()¶

    • Performs binary search on a sorted List.
    • Returns the index of the search key if it is found, otherwise returns -1.

    ✅ Example:

    import java.util.*;

public class BinarySearchExample {
    public static void main(String[] args) {
        List<Integer> numbers = Arrays.asList(1, 3, 5, 7, 9);
        Collections.sort(numbers);   // Ensure the list is sorted
        int index = Collections.binarySearch(numbers, 5);
        System.out.println("Index of 5: " + index);  // Output: Index of 5: 2
    }
}

    ✅ Binary Search on Custom Objects:

    Collections.binarySearch(list, target, Comparator.comparing(MyClass::getField));

    🔥 3. Synchronized Collection – Collections.synchronizedList()¶

    • Creates thread-safe versions of non-thread-safe collections.
    • Helps prevent ConcurrentModificationException in multi-threaded environments.

    ✅ Example:

    import java.util.*;

public class SyncListExample {
    public static void main(String[] args) {
        List<String> list = Collections.synchronizedList(new ArrayList<>());
        list.add("Apple");
        list.add("Banana");
        System.out.println(list);
    }
}

    ✅ Usage with Iterators (to avoid race conditions):

    synchronized(list) {
    Iterator<String> it = list.iterator();
    while (it.hasNext()) {
        System.out.println(it.next());
    }
}

    ⚡ 3. Additional Utility Methods in Collections Class¶

    🔥 Method ✅ Description 🔍 Usage
    reverse(List) Reverses the order of elements in a list. Collections.reverse(list)
    shuffle(List) Randomly shuffles the list elements. Collections.shuffle(list)
    max(Collection) Returns the maximum element. Collections.max(list)
    min(Collection) Returns the minimum element. Collections.min(list)
    fill(List, obj) Replaces all elements with the specified obj. Collections.fill(list, "X")
    nCopies(int, obj) Creates immutable list with N copies. Collections.nCopies(5, "Hi")
    disjoint(Collection, C2) Returns true if collections have no elements in common. Collections.disjoint(list1, list2)

    🚀 4. Best Practices and Tips¶

    • ✅ Use Collections.unmodifiableList() for read-only collections.
    • ✅ Leverage synchronizedList() in multi-threaded environments to prevent race conditions.
    • ✅ Prefer List.copyOf() (Java 10+) over Collections.unmodifiableList() for better performance.
    • ✅ Use binarySearch() only on sorted collections for accurate results.
    • ✅ Choose parallel sorting (Arrays.parallelSort()) for better performance on large data sets.

    ✅ Key Takeaways:¶

    • Collections.sort() → Sorts list in ascending order by default.
    • Collections.binarySearch() → Performs efficient binary search on sorted collections.
    • Collections.synchronizedList() → Creates a thread-safe list wrapper.
    • Use utility methods for efficient operations on collections with minimal code.
