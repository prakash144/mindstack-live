    🚀 Comparison & Sorting in Java¶

    ✅ 1. What is Comparison & Sorting?¶

    • Comparison: The process of evaluating two objects based on a defined ordering logic.
    • Sorting: The process of arranging elements in a specific order (ascending/descending).
    • Java provides two main interfaces for comparison:
      • Comparable → Natural ordering.
      • Comparator → Custom ordering.

    🔥 2. Comparable vs Comparator – Key Differences¶

    🔥 Feature ✅ Comparable 🚀 Comparator
    Package java.lang java.util
    Purpose Defines the natural ordering Defines custom ordering
    Method compareTo(Object o) compare(Object o1, Object o2)
    Syntax class Student implements Comparable<Student> class AgeComparator implements Comparator<Student>
    Modification Modifies the class itself Used outside the class
    Multiple Sorting ❌ Not supported ✅ Multiple sorting logics supported
    Used in Collections.sort(list) Collections.sort(list, comparator)

    🚀 3. Comparable Interface – Natural Ordering¶

    • Definition: Provides one sorting logic.
    • Method: int compareTo(T o) → Compares this object with the specified object.
    • Returns:
      • 0 → if both objects are equal.
      • < 0 → if this object is less than the specified object.
      • > 0 → if this object is greater than the specified object.

    ✅ Example with Comparable:¶

    import java.util.*;

class Student implements Comparable<Student> {
    int id;
    String name;

    public Student(int id, String name) {
        this.id = id;
        this.name = name;
    }

    @Override
    public int compareTo(Student s) {
        return this.name.compareTo(s.name);  // Natural ordering by name
    }

    public String toString() {
        return id + " - " + name;
    }
}

public class Main {
    public static void main(String[] args) {
        List<Student> students = new ArrayList<>();
        students.add(new Student(1, "Alice"));
        students.add(new Student(2, "Bob"));
        students.add(new Student(3, "Charlie"));

        Collections.sort(students);  // Sorting using Comparable
        System.out.println(students);
    }
}

    🔥 4. Comparator Interface – Custom Ordering¶

    • Definition: Provides multiple sorting logics.
    • Method: int compare(T o1, T o2) → Compares two different objects.
    • Returns:
      • 0 → if both objects are equal.
      • < 0 → if o1 is less than o2.
      • > 0 → if o1 is greater than o2.

    ✅ Example with Comparator:¶

    import java.util.*;

class Student {
    int id;
    String name;

    public Student(int id, String name) {
        this.id = id;
        this.name = name;
    }

    public String toString() {
        return id + " - " + name;
    }
}

class NameComparator implements Comparator<Student> {
    public int compare(Student s1, Student s2) {
        return s1.name.compareTo(s2.name);  // Custom ordering by name
    }
}

class IdComparator implements Comparator<Student> {
    public int compare(Student s1, Student s2) {
        return Integer.compare(s1.id, s2.id);  // Custom ordering by ID
    }
}

public class Main {
    public static void main(String[] args) {
        List<Student> students = new ArrayList<>();
        students.add(new Student(3, "Charlie"));
        students.add(new Student(1, "Alice"));
        students.add(new Student(2, "Bob"));

        Collections.sort(students, new NameComparator());  // Sorting by Name
        System.out.println("By Name: " + students);

        Collections.sort(students, new IdComparator());    // Sorting by ID
        System.out.println("By ID: " + students);
    }
}

    🚀 5. When to Use Comparable vs Comparator?¶

    ✅ Use Comparable: - When natural ordering is required (e.g., sorting by name, id). - For default sorting logic within the class itself. - Suitable for single sorting criteria.

    🚀 Use Comparator: - When custom ordering is required. - For multiple sorting logics (e.g., sorting by age, salary, etc.). - Ideal for external sorting and when the class doesn’t implement Comparable.

    🔥 6. Best Practices for Comparison & Sorting¶

    • Use Comparator.comparing() → When using Java 8+ for simpler syntax.
    • Use lambda expressions → To reduce boilerplate code.
    • Use Collections.sort() or List.sort() → When dealing with lists.
    • Use TreeSet or TreeMap → For automatic sorting by natural/comparator order.
    • Handle null values carefully → To avoid NullPointerException during sorting.
    • Override equals() and hashCode() → When comparing custom objects.

    ✅ 7. Key Takeaways¶

    • Comparable → For natural ordering within the class (single sorting logic).
    • Comparator → For custom ordering outside the class (multiple sorting logics).
    • Use Java 8+ features for cleaner and efficient sorting.
    • Apply sorting on collections using Collections.sort() or List.sort().

    🚀 Mastering comparison and sorting is essential for writing efficient and maintainable Java applications.
