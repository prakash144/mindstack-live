    üî• Set Implementations in Java¶

    üöÄ 1. What is a Set?¶

    • A Set is a collection that cannot contain duplicate elements.
    • It models the mathematical set abstraction.
    • No defined order (except LinkedHashSet and TreeSet which have specific ordering).
    • Part of the Java Collections Framework.

    üî• 2. Key Set Implementations¶

    üî• Implementation ‚öôÔ∏è Ordering ‚úÖ Duplicates Allowed ‚ö° Performance üìå Use Cases
    HashSet ‚ùå No ordering ‚ùå No ‚úÖ Fast (O(1) for add, remove, contains) When ordering is not required
    LinkedHashSet ‚úÖ Insertion-order maintained ‚ùå No ‚ö° Slightly slower than HashSet When insertion order is important
    TreeSet ‚úÖ Sorted in natural order ‚ùå No üî• O(log n) for operations When sorted set is required

    üî• 3. HashSet¶

    • Unordered collection of unique elements.
    • Uses a HashMap internally for storage.
    • Offers constant-time performance for basic operations.
    • Allows null values.
    • Not synchronized ‚Üí Use Collections.synchronizedSet() for thread safety.

    ‚úÖ Code Example:

    import java.util.HashSet;

public class HashSetExample {
    public static void main(String[] args) {
        HashSet<String> set = new HashSet<>();
        set.add("Apple");
        set.add("Banana");
        set.add("Apple");  // Duplicate ignored
        System.out.println(set);  // Output: [Apple, Banana]
    }
}

    üî• 4. LinkedHashSet¶

    • Preserves insertion order.
    • Uses a combination of HashMap and doubly linked list.
    • Faster iteration compared to HashSet due to linked list.
    • Allows null values.

    ‚úÖ Code Example:

    import java.util.LinkedHashSet;

public class LinkedHashSetExample {
    public static void main(String[] args) {
        LinkedHashSet<Integer> set = new LinkedHashSet<>();
        set.add(10);
        set.add(20);
        set.add(10);  // Duplicate ignored
        System.out.println(set);  // Output: [10, 20]
    }
}

    üî• 5. TreeSet¶

    • Sorted set based on NavigableSet interface.
    • Implements Red-Black Tree internally ‚Üí maintains ascending order.
    • Operations take O(log n) time.
    • Does not allow null values.

    ‚úÖ Code Example:

    import java.util.TreeSet;

public class TreeSetExample {
    public static void main(String[] args) {
        TreeSet<String> set = new TreeSet<>();
        set.add("Orange");
        set.add("Apple");
        set.add("Banana");
        System.out.println(set);  // Output: [Apple, Banana, Orange]
    }
}

    ‚úÖ 7. Best Practices & Tips¶

    • Use HashSet for fast performance and when ordering is not required.
    • Use LinkedHashSet when you need consistent iteration order.
    • Use TreeSet when you need a sorted collection.
    • For thread-safety, wrap them with Collections.synchronizedSet().

    ‚úÖ Thread-safe example:

    import java.util.Collections;
import java.util.Set;
import java.util.HashSet;

public class ThreadSafeSet {
    public static void main(String[] args) {
        Set<String> set = Collections.synchronizedSet(new HashSet<>());
        set.add("Java");
        set.add("Python");
        System.out.println(set);
    }
}

    üöÄ Key Takeaways:¶

    • HashSet ‚Üí Fast, unordered, allows null values.
    • LinkedHashSet ‚Üí Ordered by insertion, slightly slower than HashSet.
    • TreeSet ‚Üí Sorted, O(log n) performance, no null values.
    • Choose the appropriate Set implementation based on performance, ordering, and sorting needs.
