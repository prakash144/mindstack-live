    🛠️ Collection Interfaces in Java¶

    🚀 1. Introduction to Collection Framework¶

    • Java Collection Framework provides a set of interfaces and classes to store, manipulate, and retrieve data efficiently.
    • Interfaces define the abstract data structures, while classes provide concrete implementations.
    • Collections are part of the java.util package.

    🔥 2. Key Collection Interfaces¶

    🔥 Interface ✅ Definition ⚡ Key Implementations
    List<E> Ordered collection, allows duplicates ArrayList, LinkedList, Vector, Stack
    Set<E> Unordered collection, no duplicates allowed HashSet, LinkedHashSet, TreeSet
    Map<K, V> Key-Value pairs, unique keys HashMap, LinkedHashMap, TreeMap, ConcurrentHashMap
    Queue<E> FIFO (First In, First Out) structure LinkedList, PriorityQueue, ArrayDeque
    Deque<E> Double-ended queue (FIFO + LIFO) ArrayDeque, LinkedList

    🔥 3. List Interface¶

    • Ordered collection allowing duplicates.
    • Elements can be accessed by index.
    • Allows null values.

    ✅ Key Methods:

    List<String> list = new ArrayList<>();
list.add("A");           // Add element
list.get(0);              // Get element at index
list.remove("A");         // Remove element
list.set(0, "B");         // Replace element at index
list.size();               // Get list size

    ✅ Common Implementations: - ArrayList: Dynamic array, fast random access, slower insertion/deletion. - LinkedList: Doubly linked list, faster insertion/deletion, slower random access. - Vector: Synchronized, slower than ArrayList. - Stack: LIFO (Last In, First Out) structure.

    🔥 4. Set Interface¶

    • Unordered collection with unique elements.
    • No duplicates allowed.

    ✅ Key Methods:

    Set<String> set = new HashSet<>();
set.add("A");           // Add element
set.remove("A");        // Remove element
set.contains("A");      // Check existence
set.size();               // Get size

    ✅ Common Implementations: - HashSet: Unordered, fast operations, backed by a HashMap. - LinkedHashSet: Ordered by insertion, maintains insertion order. - TreeSet: Sorted collection, backed by a Red-Black tree.

    🔥 5. Map Interface¶

    • Key-Value pairs collection.
    • Unique keys, multiple values can map to the same key.
    • No direct inheritance from Collection interface.

    ✅ Key Methods:

    Map<String, Integer> map = new HashMap<>();
map.put("A", 1);          // Add key-value pair
map.get("A");             // Get value by key
map.remove("A");          // Remove key-value pair
map.containsKey("A");     // Check for key
map.size();                // Get size

    ✅ Common Implementations: - HashMap: Unordered, allows null keys and values. - LinkedHashMap: Ordered by insertion, maintains order. - TreeMap: Sorted by keys, uses Red-Black tree. - ConcurrentHashMap: Thread-safe, allows concurrent access.

    🔥 6. Queue Interface¶

    • FIFO (First In, First Out) data structure.
    • Allows null values (except PriorityQueue).

    ✅ Key Methods:

    Queue<String> queue = new LinkedList<>();
queue.add("A");            // Add element
queue.poll();               // Remove head
queue.peek();               // Get head without removing
queue.size();                // Get size

    ✅ Common Implementations: - LinkedList: Implements both Queue and Deque. - PriorityQueue: Elements are ordered based on natural ordering or custom comparator. - ArrayDeque: Resizable array implementation.

    🌟 7. Differences Between List, Set, Map, and Queue¶

    ⚙️ Feature 📚 List 🔥 Set 🚀 Map 🔁 Queue
    Order Maintains insertion order No guaranteed order No guaranteed order FIFO order
    Duplicates Allows duplicates No duplicates Keys must be unique Allows duplicates
    Null values Allows null One null value allowed One null key, multiple null values One null value (except PriorityQueue)
    Key-Value pairs No No Yes No
    Best Use Case Random access, ordered list Unique elements Key-Value pairs FIFO operations

    ✅ Key Takeaways:¶

    • List: Ordered collection, allows duplicates.
    • Set: Unordered, unique elements.
    • Map: Key-value pairs, unique keys.
    • Queue: FIFO structure, ideal for scheduling.

    🚀 Mastering Collection Interfaces is essential for handling data efficiently and is frequently tested in Java interviews!
