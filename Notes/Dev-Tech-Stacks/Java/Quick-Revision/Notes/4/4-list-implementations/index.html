    üöÄ List Implementations in Java¶

    üî• 1. What is the List Interface?¶

    • List is a part of the Java Collections Framework (extends Collection interface).
    • Ordered collection of objects (maintains insertion order).
    • Allows duplicate elements.
    • Provides indexed access to elements.
    • Implementations: ArrayList, LinkedList, Vector, and Stack.

    üî• 2. Key List Implementations Comparison¶

    üî• Implementation ‚ö° Performance üîÑ Data Structure üî• Best Use Case
    ArrayList ‚úÖ Fast random access (O(1)) Dynamic array Frequent retrieval operations
    LinkedList ‚ùå Slow random access (O(n)) Doubly linked list Frequent insertion/deletion
    Vector ‚úÖ Thread-safe, slower than ArrayList Resizable array Legacy code, thread-safe operations
    Stack ‚ùå LIFO operations only Extends Vector Stack-specific operations (LIFO)

    üî• 3. ArrayList¶

    • Dynamic array implementation.
    • Provides fast random access (O(1) for get operations).
    • Not synchronized (not thread-safe).
    • Resizable: Automatically grows when capacity is exceeded.
    • Default capacity: 10 (doubles when exceeded).
    import java.util.ArrayList;

ArrayList<String> list = new ArrayList<>();
list.add("Java");
list.add("Python");
System.out.println(list.get(0));  // Output: Java

    ‚úÖ Best for: Frequent access operations.

    üî• 4. LinkedList¶

    • Doubly linked list implementation.
    • Efficient for insertions/deletions (O(1)).
    • Slower random access (O(n)).
    • Implements Queue and Deque interfaces.
    import java.util.LinkedList;

LinkedList<String> list = new LinkedList<>();
list.add("Apple");
list.addFirst("Orange");
System.out.println(list);  // Output: [Orange, Apple]

    ‚úÖ Best for: Frequent insert/delete operations.

    üî• 5. Vector¶

    • Legacy class (since Java 1.0).
    • Synchronized (thread-safe but less efficient).
    • Resizable array (doubles in size when full).
    • Slower than ArrayList due to synchronization overhead.
    import java.util.Vector;

Vector<String> vector = new Vector<>();
vector.add("Java");
vector.add("C++");
System.out.println(vector);  // Output: [Java, C++]

    ‚úÖ Best for: Legacy systems requiring thread-safe operations.

    üî• 6. Stack¶

    • LIFO (Last-In-First-Out) data structure.
    • Extends Vector (inherits synchronization).
    • Push, pop, peek operations.
    • Slower than Deque for stack operations.
    import java.util.Stack;

Stack<Integer> stack = new Stack<>();
stack.push(10);
stack.push(20);
System.out.println(stack.pop());  // Output: 20

    ‚úÖ Best for: LIFO-specific operations.

    üî• 7. Key Differences and Use Cases¶

    • ArrayList ‚ûù Best for random access, frequent retrieval.
    • LinkedList ‚ûù Best for frequent insert/delete operations.
    • Vector ‚ûù Legacy, thread-safe but less efficient.
    • Stack ‚ûù Best for LIFO-based operations.

    ‚úÖ Tip: Prefer ArrayList over Vector for non-thread-safe operations due to better performance. ‚úÖ Tip: Use Deque over Stack for stack operations due to better performance in modern Java.
