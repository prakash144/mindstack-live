    🔁 Java Operators – Revision Notes¶

    ✅ 1. Introduction to Operators¶

    • Operators are symbols used to perform operations on variables and values.
    • Java provides several types of operators categorized by their functionality.
    • Syntax: result = operand1 operator operand2;

    ➗ 2. Arithmetic Operators¶

    Used for basic mathematical operations.

    Operator Description Example Result
    + Addition int sum = 5 + 3; sum = 8
    - Subtraction int diff = 9 - 4; diff = 5
    * Multiplication int prod = 3 * 4; prod = 12
    / Division int div = 10 / 2; div = 5
    % Modulus (remainder) int rem = 10 % 3; rem = 1

    💡 Key Points: - Integer Division: Division of two integers gives an integer result (fraction truncated). - Modulo Operator: Returns the remainder after division.

    🔍 3. Relational (Comparison) Operators¶

    Used to compare values and return a boolean result (true or false).

    Operator Description Example Result
    == Equal to 5 == 5 true
    != Not equal to 5 != 4 true
    > Greater than 10 > 5 true
    < Less than 3 < 7 true
    >= Greater than or equal 6 >= 6 true
    <= Less than or equal 4 <= 5 true

    💡 Key Points: - Used in control flow statements (if, while) and loops. - Comparison results in boolean output only.

    🔥 4. Logical Operators¶

    Used to combine multiple boolean expressions.

    Operator Description Example Result
    && Logical AND (5 > 3) && (8 > 4) true
    || Logical OR (5 > 3) || (8 < 4) true
    ! Logical NOT !(5 > 3) false

    💡 Key Points: - && → Returns true if both conditions are true. - || → Returns true if at least one condition is true. - ! → Negates the boolean value. - Short-circuiting: - For &&, if the first condition is false, the second is not evaluated. - For ||, if the first condition is true, the second is not evaluated.

    ⚙️ 5. Bitwise Operators¶

    Used to manipulate individual bits of integer values.

    Operator Description Example Result
    & Bitwise AND 5 & 3 → 101 & 011 001 (1)
    | Bitwise OR 5 | 3 → 101 | 011 111 (7)
    ^ Bitwise XOR 5 ^ 3 → 101 ^ 011 110 (6)
    ~ Bitwise Complement ~5 → ~00000101 11111010 (-6)
    << Left Shift 5 << 1 → 00000101 << 1 00001010 (10)
    >> Right Shift 5 >> 1 → 00000101 >> 1 00000010 (2)
    >>> Unsigned Right Shift -5 >>> 1 Fills with 0 bits

    💡 Key Points: - Left Shift (<<) → Multiplies by 2^n. - Right Shift (>>) → Divides by 2^n. - Unsigned Right Shift (>>>) → Fills 0 for positive and negative numbers. - Use Cases: - Cryptography, compression, low-level programming.

    🔥 6. Assignment Operators¶

    Used to assign values to variables.

    Operator Description Example Equivalent To
    = Assign a = 5 a = 5
    += Add and assign a += 3 a = a + 3
    -= Subtract and assign a -= 2 a = a - 2
    *= Multiply and assign a *= 4 a = a * 4
    /= Divide and assign a /= 2 a = a / 2
    %= Modulus and assign a %= 3 a = a % 3

    💡 Key Points: - Compound operators reduce verbosity. - Improves code readability and efficiency.

    🎯 7. Ternary Operator¶

    • A shorthand for if-else statements.
    • Syntax: variable = (condition) ? expr1 : expr2;
    • Example:
    int a = 10, b = 5;
int max = (a > b) ? a : b;  // max = 10
    • ✅ Use Cases:
      • Simple condition checking.
      • Code simplification.

    🔥 8. instanceof Operator¶

    • Used to check object type at runtime.
    • Syntax: object instanceof ClassName
    • Example:
    String str = "Java";
System.out.println(str instanceof String);  // true

    💡 Key Points: - Returns true if the object is an instance of the specified class or subclass. - Commonly used for type checking in inheritance and polymorphism.

    ✅ 9. Key Takeaways¶

    • Understand operator precedence to avoid unexpected results.
    • Use bitwise operators carefully in performance-critical applications.
    • Use ternary operators for concise code but avoid nested ternaries for readability.
    • Leverage instanceof in polymorphic code for safe type casting.
