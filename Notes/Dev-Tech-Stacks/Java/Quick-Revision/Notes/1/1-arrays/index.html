    üéØ Arrays in Java¶

    ‚úÖ 1. Introduction to Arrays¶

    • Definition:
      • An array is a collection of variables of the same type stored in contiguous memory locations.
    • Characteristics:
      • Fixed-size: Once declared, the size cannot be changed.
      • Index-based: Index starts at 0.
      • Efficient data retrieval: Faster access by index.

    üî• 2. Declaration, Initialization & Iteration¶

    üìå Declaration:¶

    // Syntax: datatype[] arrayName;
int[] numbers;   // Recommended syntax
int numbers[];   // Valid but less preferred

    üìå Initialization:¶

    int[] nums = new int[5];          // Array of size 5 (default values = 0)
int[] primes = {2, 3, 5, 7, 11};  // Array with initialized values
    • Default values:
      • int ‚Üí 0
      • float ‚Üí 0.0
      • boolean ‚Üí false
      • String ‚Üí null

    üìå Iteration Methods:¶

    • For Loop:
    int[] arr = {1, 2, 3, 4};
for (int i = 0; i < arr.length; i++) {
    System.out.println(arr[i]);
}
    • Enhanced For Loop (for-each):
    for (int num : arr) {
    System.out.println(num);
}
    • Using Arrays.stream() (Java 8+):
    Arrays.stream(arr).forEach(System.out::println);

    üî• 3. Multidimensional Arrays¶

    üìå Declaration & Initialization:¶

    int[][] matrix = new int[3][4];     // 3x4 matrix
int[][] grid = {{1, 2}, {3, 4}, {5, 6}}; // Initialization

    üìå Iteration:¶

    for (int i = 0; i < matrix.length; i++) {
    for (int j = 0; j < matrix[i].length; j++) {
        System.out.print(matrix[i][j] + " ");
    }
    System.out.println();
}
    • Jagged Arrays: (rows can have different column sizes)
    int[][] jagged = new int[3][];
jagged[0] = new int[2];
jagged[1] = new int[3];
jagged[2] = new int[1];

    ‚öôÔ∏è 4. Array Operations & Methods¶

    ‚úÖ Sorting an Array:¶

    int[] numbers = {5, 1, 4, 2, 3};
Arrays.sort(numbers);  // Ascending order
    • Descending Sort:
    Arrays.sort(numbers, Collections.reverseOrder());

    ‚úÖ Searching in an Array:¶

    int[] nums = {2, 4, 6, 8, 10};
int index = Arrays.binarySearch(nums, 6);  // Returns index of 6
    • If the element is not found ‚Üí Returns -(insertion_point) - 1

    ‚úÖ Filling an Array:¶

    int[] arr = new int[5];
Arrays.fill(arr, 7);    // Fills all elements with 7

    ‚úÖ Copying Arrays:¶

    int[] original = {1, 2, 3};
int[] copy = Arrays.copyOf(original, 5);   // Copy with size 5

    ‚úÖ Comparing Arrays:¶

    int[] arr1 = {1, 2, 3};
int[] arr2 = {1, 2, 3};
System.out.println(Arrays.equals(arr1, arr2));  // true

    ‚úÖ Converting Arrays to Strings:¶

    System.out.println(Arrays.toString(arr));

    üöÄ 5. Best Practices & Tips¶

    • ‚úÖ Use Arrays.stream() and forEach() for concise iteration (Java 8+).
    • ‚úÖ Prefer List (ArrayList) over raw arrays for dynamic sizing.
    • ‚úÖ Use System.arraycopy() or Arrays.copyOf() for efficient copying.
    • ‚úÖ Be aware of ArrayIndexOutOfBoundsException while accessing elements.

    üî• 6. Time & Space Complexities¶

    Operation Time Complexity Space Complexity
    Access by index O(1) O(1)
    Insertion at end O(1) (amortized) O(1)
    Insertion at middle O(n) O(1)
    Deletion at middle O(n) O(1)
    Search (Linear) O(n) O(1)
    Binary Search O(log n) (Sorted) O(1)
    Sorting O(n log n) O(n) (merge sort)

    ‚úÖ Key Takeaways:¶

    • Arrays in Java are fixed-size and store homogeneous data.
    • Multidimensional arrays allow storing data in matrix form.
    • Use Arrays class methods for efficient operations (e.g., sort(), binarySearch(), fill()).
    • Time complexity for accessing elements is O(1); searching takes O(n) in unsorted arrays.
    • Prefer ArrayList for dynamic sizing in real-world applications.
