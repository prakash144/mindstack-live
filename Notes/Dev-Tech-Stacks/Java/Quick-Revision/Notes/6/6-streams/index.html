    🚀 Java Streams API – Revision Notes¶

    🌊 What is the Java Streams API?¶

    • Introduced in Java 8 for functional-style operations on collections.
    • Streams allow declarative data processing: think of what to do, not how to do it.
    • Operates on data pipelines, not data structures.
    • Supports parallel execution for performance improvements.

    🛠️ Stream Pipeline¶

    A pipeline is a sequence of stream operations:

    Collection.stream()
          .intermediateOp1()
          .intermediateOp2()
          ...
          .terminalOp();

    🔁 Intermediate Operations (Lazy)¶

    • Don't execute until a terminal operation is called.
    • Return a new Stream each time.
    Operation Purpose Example
    map() Transform each element stream.map(String::toUpperCase)
    filter() Select elements matching a condition stream.filter(s -> s.length() > 3)
    sorted() Sort elements stream.sorted()
    distinct() Remove duplicates stream.distinct()
    limit() Limit the size of the stream stream.limit(5)

    🏁 Terminal Operations (Eager)¶

    • Trigger the pipeline and produce a result or side-effect.
    • Once called, the stream is consumed and cannot be reused.
    Operation Purpose Example
    forEach() Iterate and apply action stream.forEach(System.out::println)
    collect() Convert to List, Set, Map, etc. stream.collect(Collectors.toList())
    reduce() Aggregate elements into a single value stream.reduce(0, Integer::sum)
    count() Count elements stream.count()
    anyMatch() Check if any match a predicate stream.anyMatch(s -> s.isEmpty())

    ⚙️ Key Functional Interfaces Used¶

    • Function<T, R> → used in map()
    • Predicate<T> → used in filter()
    • Consumer<T> → used in forEach()
    • BinaryOperator<T> → used in reduce()

    💡 Deep Dive: reduce() Explained¶

    Optional<Integer> result = Stream.of(1, 2, 3, 4)
    .reduce((a, b) -> a + b); // Optional[10]
    • Used for aggregation, like sum, min, max.
    • Can be used with identity value and accumulator function.
    • Parallel reduction possible via Stream.parallel()

    🧠 Pro Tips for Experienced Engineers¶

    • ✅ Prefer Streams over loops for readability and maintainability.
    • 🔄 Combine filter → map → collect for cleaner code.
    • 🧵 Use .parallelStream() cautiously — test performance & thread-safety.
    • 🚫 Avoid stateful operations (e.g., shared mutable state) inside streams.
    • 🧹 Use Collectors.groupingBy() and Collectors.partitioningBy() for data grouping needs.

    📌 Summary Cheat Sheet¶

    List<String> names = Arrays.asList("Alice", "Bob", "Charlie");
List<String> upper = names.stream()
                          .filter(n -> n.length() > 3)
                          .map(String::toUpperCase)
                          .collect(Collectors.toList());
System.out.println(upper); // [ALICE, CHARLIE]

    ➡️ Streams = Clean, functional, parallel-friendly code for processing collections.

    📚 Explore More¶

    • Collectors.toMap() for building maps.
    • flatMap() for flattening nested streams.
    • Custom collectors using Collector.of().
    • Performance tuning with Spliterators.
