    🔥 Lambda Expressions in Java¶

    🚀 1. What are Lambda Expressions?¶

    • 🧠 Lambda Expressions are a feature introduced in Java 8.
    • Enable treating functionality as method arguments or treating code as data.
    • Support functional programming using concise and expressive syntax.
    • Eliminate boilerplate code used in anonymous inner classes.

    🧩 2. Syntax¶

    (parameters) -> expression
(parameters) -> { statements; }

    ✅ Examples:¶

    Runnable r = () -> System.out.println("Hello from Lambda!");
Comparator<Integer> comp = (a, b) -> Integer.compare(a, b);

    🔍 3. Key Characteristics¶

    • 🔹 Can be used only with functional interfaces (interfaces with a single abstract method).
    • 🔹 Can capture effectively final variables from the enclosing scope.
    • 🔹 Promotes cleaner and readable code.

    🛠️ 4. Functional Interface Examples¶

    @FunctionalInterface
interface MyFunctionalInterface {
    void execute();
}
    • ✅ Predefined interfaces in java.util.function package:
      • Predicate<T> – returns boolean
      • Function<T, R> – takes T, returns R
      • Supplier<T> – provides object of type T
      • Consumer<T> – consumes T with no return

    📚 5. Use Cases in Real-World Applications¶

    • 🔁 Iteration with forEach(): java list.forEach(item -> System.out.println(item));
    • 📊 Filtering and mapping in Streams: java List<String> filtered = names.stream() .filter(name -> name.startsWith("A")) .collect(Collectors.toList());
    • 🔧 Event handling in GUI frameworks
    • 🧪 Unit testing: mocking and stubbing with frameworks like Mockito

    ⚙️ 6. Advanced Considerations for Experienced Developers¶

    • 🔍 Debugging is less intuitive—use named methods when debugging complex lambdas.
    • 🧵 Thread safety: avoid mutating shared state within lambdas used in parallel streams.
    • 🔁 Recursive behavior not directly supported—workaround via static helper methods.
    • 🔒 Checked exceptions must be handled inside the lambda or rethrown indirectly.

    ✅ 7. Benefits¶

    • 🚀 Concise and expressive code
    • 🔄 Enhanced use of Stream API and Collections API
    • 🔧 Better suited for functional pipelines

    ⚠️ 8. Limitations¶

    • ❌ Can’t have multiple abstract methods in the target interface
    • ❗ Overuse can make code less readable for large teams

    🧠 9. Best Practices¶

    • ✅ Use for small, simple operations
    • ✅ Use method references (ClassName::method) when possible for readability
    • ❌ Avoid long or nested lambda expressions—prefer named functions

    📌 10. Quick Summary Table¶

    🔧 Feature ✅ Supported In Lambda
    Access outer variables ✅ If effectively final
    Multiple parameters ✅
    Checked exceptions ⚠️ Manual handling
    Functional interface use ✅ Required
    Return statements ✅ Optional for 1-liners
    Recursion ❌ Use workaround

    🧩 Pro Tip: Combine Streams + Lambdas to express powerful data transformations elegantly.
