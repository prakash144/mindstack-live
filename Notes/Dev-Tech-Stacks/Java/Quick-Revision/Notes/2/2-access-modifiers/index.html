    üî• Access Modifiers in Java¶

    üöÄ 1. What are Access Modifiers?¶

    • Access Modifiers control the visibility and accessibility of classes, methods, variables, and constructors.
    • They define which parts of the code can be accessed from other classes, packages, or subclasses.
    • Improper use can lead to tight coupling, reduced encapsulation, and security issues.

    üî• 2. Types of Access Modifiers¶

    üî• Modifier üåê Scope ‚úÖ Accessible From üîí Usage in Classes
    public Anywhere Same class, same package, different package, subclass ‚úÖ Allowed
    protected Same package + Subclasses Same class, same package, different package via inheritance ‚ùå Not allowed at class level
    default (package-private) Same package only Same class, same package only ‚úÖ Allowed
    private Within the same class only Same class only ‚úÖ Allowed

    ‚úÖ 3. Detailed Explanation with Examples¶

    üî• 1. public Access Modifier¶

    • Scope: Accessible from anywhere.
    • Usage: Best for utility classes, constants, and APIs meant for global access.
    • Example:
    public class Vehicle {
    public String brand = "Toyota";
    public void display() {
        System.out.println("Brand: " + brand);
    }
}
    • ‚úÖ Accessible from any package or class.

    üî• 2. protected Access Modifier¶

    • Scope: Accessible in the same package and by subclasses (even in different packages).
    • Usage: Ideal for methods meant for inheritance or extensibility.
    • Example:
    package com.example;

public class Car {
    protected String model = "Sedan";

    protected void showModel() {
        System.out.println("Model: " + model);
    }
}

    package com.inherited;
import com.example.Car;

public class SUV extends Car {
    public void display() {
        showModel();  // Accessible due to inheritance
    }
}
    • ‚úÖ Accessible in subclasses across different packages.

    üî• 3. default (Package-Private) Access Modifier¶

    • Scope: Accessible within the same package only.
    • Usage: Suitable for package-level utilities.
    • Example:
    class Vehicle {  // Default access
    String type = "Car";  // Default access

    void displayType() {   // Default access
        System.out.println("Type: " + type);
    }
}
    • ‚ùå Not accessible outside the package.

    üî• 4. private Access Modifier¶

    • Scope: Accessible within the same class only.
    • Usage: Ideal for data encapsulation and internal implementation details.
    • Example:
    public class Person {
    private String name = "John";

    private void display() {
        System.out.println("Name: " + name);
    }
}
    • ‚ùå Not accessible outside the class.

    üî• 4. Key Takeaways and Best Practices¶

    • Use private for fields and methods that should not be accessed directly.
    • Use protected for inheritance-based extensibility.
    • Use public only for APIs or methods meant to be accessed globally.
    • Favor encapsulation: Keep fields private and provide public getter/setter methods.
    • Minimize protected access to reduce unintentional misuse.
    • Default access is package-private, suitable for classes only meant to be used within the same package.

    üöÄ 5. Interview Tips & Key Points¶

    • Inheritance & Protected: Methods with protected access can be accessed in child classes from different packages.
    • Default Access: Often a common source of confusion‚Äîonly accessible within the same package.
    • Encapsulation Principle: private variables with public getters/setters maintain control over the data.
    • Memory & Security: Use private access for sensitive data (e.g., passwords, IDs).
    • Avoid exposing internals: Favor access restriction over excessive public access.

    ‚úÖ Mastering access modifiers is essential for writing secure, modular, and maintainable Java applications.
