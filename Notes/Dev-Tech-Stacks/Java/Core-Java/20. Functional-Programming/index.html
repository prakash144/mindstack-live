    üöÄ Must-Know Functional Programming Concepts in Java (for Spring Boot & Microservices)¶

    1Ô∏è‚É£ Functional Interfaces (java.util.function)¶

    Single Abstract Method (SAM) interfaces used in functional programming. Java provides built-in functional interfaces:

    Functional Interface Method Purpose
    Predicate<T> boolean test(T t) Used for conditional checks (returns true/false).
    Function<T, R> R apply(T t) Converts T to R.
    Consumer<T> void accept(T t) Performs an action on T.
    Supplier<T> T get() Provides a value of type T. 
    Predicate<Integer> isEven = n -> n % 2 == 0;
System.out.println(isEven.test(10)); // true

    üîπ Example (Spring Boot use-case: Filtering API Requests)¶

    Predicate<String> isValidRequest = req -> req.startsWith("VALID_");
System.out.println(isValidRequest.test("VALID_ORDER")); // true

    2Ô∏è‚É£ Lambda Expressions (üìå Key to Functional Programming)¶

    Anonymous function for concise code. Syntax: (parameters) -> { body }

    Function<String, Integer> length = str -> str.length();
System.out.println(length.apply("Hello")); // 5

    üîπ Example (Microservice Response Transformation)¶

    Function<Order, OrderDTO> orderMapper = order -> new OrderDTO(order.getId(), order.getTotal());

    ‚úÖ Spring Boot Use-case: Mapping Entities to DTOs in Service Layer

    3Ô∏è‚É£ Method References (üöÄ Shortcut for Lambdas)¶

    Refers to existing methods instead of using lambdas Syntax: ClassName::methodName

    Function<String, Integer> length = String::length;
System.out.println(length.apply("World")); // 5

    üîπ Example (Logging Requests in a Microservice)¶

    Consumer<String> logger = System.out::println;
logger.accept("Request received...");

    ‚úÖ Spring Boot Use-case: Logging incoming API requests using method references.

    4Ô∏è‚É£ Stream API (üî• Functional Operations on Collections)¶

    Efficient processing of collections.

    List<String> names = List.of("Alice", "Bob", "Charlie");
names.stream().filter(name -> name.startsWith("A")).forEach(System.out::println);

    Output: Alice - ‚úÖ Key Stream Methods: * filter(Predicate) ‚Üí Filters elements * map(Function) ‚Üí Transforms elements * forEach(Consumer) ‚Üí Iterates over elements * collect(Collectors.toList()) ‚Üí Collects result into a list

    üîπ Example (Processing API Responses)¶

    List<Order> orders = orderRepository.findAll();
List<OrderDTO> orderDTOs = orders.stream()
    .filter(order -> order.getTotal() > 100)
    .map(order -> new OrderDTO(order.getId(), order.getTotal()))
    .collect(Collectors.toList());

    ‚úÖ Spring Boot Use-case: Processing a list of orders fetched from the database.

    5Ô∏è‚É£ Optional (üõ°Ô∏è Avoid NullPointerException)¶

    Handles missing values safely.

    Optional<String> name = Optional.ofNullable(null);
System.out.println(name.orElse("Default")); // Default
    • ‚úÖ Key Optional Methods:
    • isPresent() ‚Üí Checks if value exists
    • orElse(T other) ‚Üí Returns value or default
    • map(Function) ‚Üí Transforms value if present

    üîπ Example (Handling Null API Responses)¶

    Optional<User> user = userRepository.findById(1);
user.ifPresent(u -> System.out.println("User found: " + u.getName()));

    ‚úÖ Spring Boot Use-case: Avoid null checks in database lookups.

    6Ô∏è‚É£ Functional Composition (‚è© Chain Functions)¶

    Combining multiple functions.

    Function<Integer, Integer> square = x -> x * x;
Function<Integer, Integer> doubleIt = x -> x * 2;
System.out.println(square.andThen(doubleIt).apply(3)); // 18

    üîπ Example (Data Transformation Pipeline in Microservices)¶

    Function<String, String> trim = String::trim;
Function<String, String> toUpper = String::toUpperCase;
Function<String, String> process = trim.andThen(toUpper);

System.out.println(process.apply("  hello ")); // HELLO

    ‚úÖ Spring Boot Use-case: Processing incoming request payloads before validation.

    7Ô∏è‚É£ Higher-Order Functions (üì¢ Function as Parameter/Return)¶

    Pass functions as arguments or return them.

    static Function<Integer, Integer> multiply(int num) {
    return x -> x * num;
}
Function<Integer, Integer> triple = multiply(3);
System.out.println(triple.apply(5)); // 15

    üîπ Example (Custom Retry Mechanism in Microservices)¶

    public static <T> T retry(Supplier<T> task, int attempts) {
    for (int i = 0; i < attempts; i++) {
        try {
            return task.get();
        } catch (Exception e) {
            System.out.println("Retrying... " + (i + 1));
        }
    }
    throw new RuntimeException("Failed after retries");
}

// Usage:
Supplier<String> apiCall = () -> callExternalAPI();
String response = retry(apiCall, 3);

    ‚úÖ Spring Boot Use-case: Retrying external API calls (circuit breaker pattern).

    8Ô∏è‚É£ Currying (üîó Breaking Function into Smaller Functions)¶

    Transforms multi-argument functions into a series of functions

    Function<Integer, Function<Integer, Integer>> add = x -> y -> x + y;
System.out.println(add.apply(5).apply(3)); // 8

    üîπ Example (Configurable Discount Service in E-Commerce Microservices)¶

    Function<Double, Function<Double, Double>> discount = rate -> price -> price * (1 - rate);
Function<Double, Double> tenPercentDiscount = discount.apply(0.10);
System.out.println(tenPercentDiscount.apply(200.0)); // 180.0

    ‚úÖ Spring Boot Use-case: Applying different discount strategies dynamically.

    9Ô∏è‚É£ Parallel Streams (‚ö° Boost Performance with Multi-threading)¶

    Runs operations in parallel for large datasets.

    List<Integer> numbers = List.of(1, 2, 3, 4, 5);
numbers.parallelStream().forEach(System.out::println);

    üîπ Example (Processing Large Order Lists in E-Commerce Microservice)¶

    List<Order> orders = orderRepository.findAll();
orders.parallelStream()
    .filter(order -> order.getTotal() > 100)
    .forEach(System.out::println);

    ‚úÖ Spring Boot Use-case: Optimizing bulk data processing in microservices.

    üîü Reactive Programming with Functional Style (Spring WebFlux)¶

    Handles async & non-blocking API calls. Mono (single value) and Flux (multiple values) from Project Reactor.

    üîπ Example (Fetching User Details from Reactive Database in Microservices)¶

    public Mono<UserDTO> getUser(Long id) {
    return userRepository.findById(id)
        .map(user -> new UserDTO(user.getId(), user.getName()));
}

    ‚úÖ Spring Boot Use-case: Building high-performance non-blocking REST APIs.
