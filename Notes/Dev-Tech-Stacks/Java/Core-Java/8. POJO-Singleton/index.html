    Java POJO, Enum, and Singleton Classes¶

    POJO (Plain Old Java Object)¶

    • Definition: A simple Java class with private fields, public getters and setters, and a no-args constructor.
    • Usage: Maps request data in a consistent format; centralizes changes to a single class.
    • Example: Public class with default constructor, fields, and getter & setter methods.

    Enum Class¶

    1. Basic Enum Class

      • Definition: Represents a fixed set of constants.
      • Example: enum Day { MONDAY, TUESDAY, WEDNESDAY }
      • Methods:
        • values(): Returns all constants as an array.
        • valueOf("CONSTANT"): Matches and returns a constant.
      • Ordinals: Each constant has an index (0 for MONDAY, etc.)

    2. Enum with Custom Values

      • Usage: Can assign custom values to each constant.
      • Constructor: Define a parameterized constructor for custom values.

    3. Method Override by Constant

      • Behavior: Constants can override methods for specific behaviors.

    4. Enum with Abstract Method

      • Usage: Abstract methods implemented by all constants.

    5. Enum Implements Interface

      • Definition: Enum can implement an interface for standardized behavior.

    Enum Benefits over Static Final Constants¶

    • Advantages: Improved readability, controlled parameter values, and type safety.

    Enum with Custom Values Example¶

    Enum representing days with working hours:

    public enum Day {
    MONDAY(8), TUESDAY(8), WEDNESDAY(8);

    private int hours;

    private Day(int hours) {
        this.hours = hours;
    }

    public int getHours() {
        return hours;
    }
}

    Final Class¶

    • Definition: A class that cannot be extended.
    • Example: final class Constants { ... }

    Singleton Class¶

    • Objective: Restrict a class to a single instance (e.g., database connection).

    • Implementations:

      1. Eager Initialization

        • Steps:
          • Create a private static instance.
          • Use a private constructor.
          • Public static method to return the instance.

      2. Lazy Initialization

        • Definition: Instance is created only when requested.
        • Drawback: Not thread-safe; two threads may create separate instances.

      3. Synchronized Method

        • Definition: Ensures only one thread can access the creation method at a time.
        • Drawback: Slower performance.

      4. Double-Checked Locking

        • Definition: Applies synchronization to a block, reducing the performance hit.
        • Issue: Cache vs. memory inconsistency, solved with the volatile keyword.

      5. Bill Pugh Solution

        • Definition: Uses a private static nested class that loads the instance only when accessed.

      6. Enum Singleton

        • Definition: Ensures a single instance per JVM by default, as enums restrict multiple instances.

    Singleton Implementation Example¶

    Singleton class for a database connection:

    public class DatabaseConnection {
    private static DatabaseConnection instance;

    private DatabaseConnection() { ... }

    public static DatabaseConnection getInstance() {
        if (instance == null) {
            synchronized(DatabaseConnection.class) {
                if (instance == null) {
                    instance = new DatabaseConnection();
                }
            }
        }
        return instance;
    }
}

    Immutable Class¶

    • Definition: Class whose state cannot be modified after creation.
    • Implementation: Use private fields, no setters, and pass copies of mutable objects.
    public final class ImmutablePerson {
    private final String name;
    private final List<String> hobbies;

    public ImmutablePerson(String name, List<String> hobbies) {
        this.name = name;
        // Creating a copy of the original list to ensure immutability
        this.hobbies = Collections.unmodifiableList(List.copyOf(hobbies));
    }

    public String getName() { return name; }
    public List<String> getHobbies() { return hobbies; }
}
    • final class and fields ensure immutability.
    • Defensive copy with unmodifiableList prevents modification of the hobbies list.
