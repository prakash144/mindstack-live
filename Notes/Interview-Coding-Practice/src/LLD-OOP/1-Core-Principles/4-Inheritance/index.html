    üß¨ Inheritance in OOP¶

    Inheritance is a key pillar of Object-Oriented Programming (OOP). It enables code reuse, supports hierarchies, and forms the basis for polymorphism.

    üìú Definition¶

    Inheritance allows a subclass (child) to inherit fields and methods from a superclass (parent). This promotes reusability and structured extension of features.

    üë®‚Äçüë©‚Äçüëß Core Terminology¶

    • üëë Superclass (Parent)

      • Provides common properties and behaviors
      • Acts as a blueprint
      • Example: class Animal

    • üßí Subclass (Child)

      • Inherits and can override or extend parent features
      • Example: class Dog extends Animal

    üîÄ Types of Inheritance (in Java)¶

    1Ô∏è‚É£ Single Inheritance¶

    One child inherits from one parent.

    class Animal {
    void sound() { System.out.println("Animal makes sound"); }
}
class Dog extends Animal {
    void bark() { System.out.println("Dog barks"); }
}

    2Ô∏è‚É£ Multilevel Inheritance¶

    Class inherits from child of another class (multi-level chain).

    class Animal {
    void eat() { System.out.println("Eating..."); }
}
class Dog extends Animal {
    void bark() { System.out.println("Barking..."); }
}
class Puppy extends Dog {
    void weep() { System.out.println("Weeping..."); }
}

    3Ô∏è‚É£ Hierarchical Inheritance¶

    Multiple child classes inherit from a single parent class.

    class Animal {
    void sound() { System.out.println("Animal sound"); }
}
class Dog extends Animal {
    void bark() { System.out.println("Dog barks"); }
}
class Cat extends Animal {
    void meow() { System.out.println("Cat meows"); }
}

    üí° Advantages¶

    • üîÅ Reusability: Avoid duplication, reuse parent code.
    • üß© Modularity: Classes become more focused and isolated.
    • üîß Extensibility: Add or override features without modifying the parent.
    • üßº Maintainability: Centralize changes in the superclass.

    üîë Key Concepts in Inheritance¶

    üîÅ Method Overriding¶

    • Subclass redefines a method from the parent class
    class Animal {
    void sound() { System.out.println("Animal sound"); }
}
class Dog extends Animal {
    @Override
    void sound() { System.out.println("Dog barks"); }
}

    ‚úÖ Rules:

    • Same name, return type, and parameters
    • Can't have stricter access modifier
    • Use @Override annotation for clarity

    üß≠ super Keyword¶

    • Access parent class members (methods/variables)
    • Call parent constructor
    class Animal {
    Animal() { System.out.println("Animal constructor"); }
}
class Dog extends Animal {
    Dog() {
        super();  // Calls Animal constructor
        System.out.println("Dog constructor");
    }
}

    üîÑ Overloading vs Overriding¶

    Feature Overloading Overriding
    Scope Same class Across parent-child classes
    Method Signature Different parameters Same method signature
    Return Type Can differ Must be same or covariant
    Access Modifier Any Cannot be more restrictive
    Inheritance Required ‚ùå ‚úÖ

    ‚ö†Ô∏è Multiple Inheritance & Diamond Problem¶

    Multiple Inheritance: A class inherits from more than one parent.

    Diamond Problem:

           A
     /   \
    B     C
     \   /
       D
    • If B and C override a method from A, what should D inherit?
    • Java disallows multiple class inheritance to avoid this.

    ‚úÖ Java uses interfaces to allow multiple inheritance safely.

    interface A { void show(); }
interface B { void display(); }
class C implements A, B {
    public void show() { ... }
    public void display() { ... }
}

    üîÑ Real Life Example¶

    // üëë Superclass
class Product {
    String name;
    String category;
    double price;

    Product(String name, String category, double price) {
        this.name = name;
        this.category = category;
        this.price = price;
    }

    void displayInfo() {
        System.out.println("Product: " + name);
        System.out.println("Category: " + category);
        System.out.println("Price: $" + price);
    }
}

// ‚ö° Electronics subclass
class Electronics extends Product {
    String brand;
    int warrantyInMonths;

    Electronics(String name, double price, String brand, int warrantyInMonths) {
        super(name, "Electronics", price);
        this.brand = brand;
        this.warrantyInMonths = warrantyInMonths;
    }

    @Override
    void displayInfo() {
        super.displayInfo();
        System.out.println("Brand: " + brand);
        System.out.println("Warranty: " + warrantyInMonths + " months");
    }
}

// üëï TShirt subclass
class TShirt extends Product {
    String size;
    String color;

    TShirt(String name, double price, String size, String color) {
        super(name, "Clothing", price);
        this.size = size;
        this.color = color;
    }

    @Override
    void displayInfo() {
        super.displayInfo();
        System.out.println("Size: " + size);
        System.out.println("Color: " + color);
    }
}

// üß™ Main class to test
public class Main {
    public static void main(String[] args) {
        Electronics laptop = new Electronics("MacBook Pro", 180000, "Apple", 24);
        TShirt tshirt = new TShirt("Round Neck Tee", 999, "L", "Blue");

        System.out.println("üñ•Ô∏è Electronics Product:");
        laptop.displayInfo();

        System.out.println("\nüëö Clothing Product:");
        tshirt.displayInfo();
    }
}

    ‚úÖ Highlights¶

    • Product is a generic superclass.
    • Electronics and TShirt are specialized subclasses extending and customizing behavior.
    • Demonstrates constructor chaining, overriding, and clean design using super.

    ‚úÖ Summary & Best Practices¶

    • üõ† Use inheritance for IS-A relationships
    • üîê Combine with access modifiers for better encapsulation
    • üéØ Favor composition over inheritance when applicable
    • ‚úÖ Always use @Override for clarity
    • üìè Keep inheritance chains short and meaningful

    "Design with intention: Inherit only to extend behavior, not to share data."
