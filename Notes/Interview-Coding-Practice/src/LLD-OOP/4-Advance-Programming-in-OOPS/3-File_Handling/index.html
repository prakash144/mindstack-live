    📂 File Handling in Java (OOP Perspective)¶

    🚀 Senior Engineer Quick Notes – Focused on clean architecture, maintainability, and real-world application.

    ✅ Why File Handling Matters in OOP¶

    • Enables data persistence outside memory.
    • Crucial for logging, config management, and data serialization.
    • Useful in LLD when handling logs, audit trails, or externalized data configs.

    🔧 Common File Operations¶

    Operation Description
    createNewFile() Creates a new file
    write() Writes data to file
    read() Reads data from file
    append() Adds new data to existing file
    delete() Removes file

    📁 Creating a File¶

    import java.io.File;
import java.io.IOException;

public class FileCreator {
    public static void main(String[] args) {
        try {
            File file = new File("example.txt");
            if (file.createNewFile()) {
                System.out.println("File created: " + file.getName());
            } else {
                System.out.println("File already exists.");
            }
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}

    ✅ Use try-catch to handle IOException during file creation.

    📚 File Class – Commonly Used Methods¶

    file.exists();             // Check if file exists
file.delete();             // Delete file
file.getAbsolutePath();    // Get path
file.canRead();            // Check read permission
file.length();             // File size in bytes

    ✍️ Writing to File (BufferedWriter)¶

    import java.io.*;

public class WriteExample {
    public static void main(String[] args) {
        try (BufferedWriter writer = new BufferedWriter(new FileWriter("log.txt"))) {
            writer.write("Hello, file!");
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}

    📖 Reading from File (BufferedReader)¶

    import java.io.*;

public class ReadExample {
    public static void main(String[] args) {
        try (BufferedReader reader = new BufferedReader(new FileReader("log.txt"))) {
            String line;
            while ((line = reader.readLine()) != null) {
                System.out.println(line);
            }
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}

    🔁 Try-With-Resources (Auto Close)¶

    try (BufferedReader br = new BufferedReader(new FileReader("config.txt"))) {
    // read lines
} catch (IOException e) {
    // handle exception
}

    🔐 Ensures all streams are closed – No leaks!

    🧾 Simple Logger Class Example¶

    import java.io.*;

public class Logger {
    private String path;

    public Logger(String path) {
        this.path = path;
        try {
            File logFile = new File(path);
            if (!logFile.exists()) logFile.createNewFile();
        } catch (IOException e) {
            System.out.println("Log file creation failed.");
        }
    }

    public void log(String message) {
        try (BufferedWriter writer = new BufferedWriter(new FileWriter(path, true))) {
            writer.write(message);
            writer.newLine();
        } catch (IOException e) {
            System.out.println("Failed to write log.");
        }
    }
}

    🔍 Add timestamps, user IDs, session info for production-grade logs.

    ⚠️ Common Pitfalls¶

    Issue Fix Recommendation
    FileNotFoundException Check file existence or path before using
    IOException Always wrap in try-catch
    Resource Leaks Use try-with-resources

    🧠 Design Takeaways for LLD¶

    • Prefer interfaces (Writer, Reader) → Decouple file access logic
    • Create abstractions like FileStorageService for maintainability
    • Always handle exceptions at boundaries → avoid leaking I/O errors upstream
    • Make file paths configurable → avoid hard-coded paths

    📌 Pro Tip: Always log exceptions into a separate error log file with context.
