    üîí Inner Classes in Java ‚Äì System Design Perspective¶

    üß© Purpose of Inner Classes¶

    • Encapsulation of helper logic relevant only within the outer class.
    • Reduces clutter and increases cohesion for logically grouped code.
    • Promotes modularity and reduces namespace pollution.

    üîó Types of Inner Classes¶

    1Ô∏è‚É£ Static Nested Class¶

    • Declared with static keyword inside outer class.
    • Does not need instance of outer class.
    • Can only access static members of outer class.
    • ‚úÖ Best when inner logic is independent of outer instance (e.g., Builder pattern).
    class Vehicle {
    static class Engine {
        void start() {
            System.out.println("Engine started");
        }
    }
}

    2Ô∏è‚É£ Non-Static (Member) Inner Class¶

    • Tied to instance of outer class.
    • Has full access (private + instance vars).
    • Needs outer instance to instantiate.
    • ‚úÖ Ideal when inner class needs access to instance-specific data.
    class Car {
    private String model = "Sedan";

    class Dashboard {
        void showModel() {
            System.out.println("Car model: " + model);
        }
    }
}

    3Ô∏è‚É£ Local Inner Class¶

    • Defined within a method or block.
    • Accesses final/effectively final local variables.
    • Scope-limited ‚Äì not visible outside method.
    • ‚úÖ Use when behavior is method-specific.
    void process() {
    final int threshold = 10;
    class Checker {
        boolean isValid(int val) {
            return val > threshold;
        }
    }
}

    4Ô∏è‚É£ Anonymous Inner Class¶

    • A one-time implementation of a class or interface.
    • Defined inline ‚Äî no class name.
    • ‚úÖ Use for short-lived behavior (e.g., event handling, strategy injection).
    Runnable r = new Runnable() {
    @Override
    public void run() {
        System.out.println("Thread running");
    }
};

    üß† Real-World Use Case: Anonymous Inner Class Overriding ArrayList to Restrict Additions to Even Numbers¶

    @Override
    public boolean add(Integer num) {
        if (num % 2 == 0) {
            return super.add(num);
        } else {
            throw new IllegalArgumentException("Only even numbers allowed");
        }
    }

    ‚úÖ When to Use Inner Classes¶

    Use Case Inner Class Type Rationale
    Class-independent helper Static Nested Doesn't need instance context
    Needs outer instance Member Inner Accesses fields of enclosing class
    Scope-limited behavior Local Inner Used inside a specific method
    One-off override or interface impl Anonymous Inner Short, custom behavior inline

    üß∞ Design Insights for System Design¶

    • Reduces visibility scope of implementation details.
    • Avoids unnecessary public classes ‚Üí keeps APIs clean.
    • Anonymous inner classes are a gateway to Strategy pattern on-the-fly.
    • Static nested classes often used in Builder or Factory patterns.

    ‚ö†Ô∏è Trade-offs¶

    • Excessive use can lead to hard-to-read nested scopes.
    • Local and anonymous inner classes cannot be reused ‚Äî trade-off between brevity and flexibility.
    • Anonymous classes pre-Java 8 can be verbose ‚Üí consider lambdas where possible.

    ‚úÖ Best Practices¶

    • Prefer static nested class if inner logic doesn‚Äôt require outer class instance.
    • Use anonymous inner classes for concise behavior (UI/event-driven code, strategy injections).
    • Avoid deep nesting (inner within inner) ‚Äî hurts readability.

    Inner classes, when applied right, are tactical tools for modularity, encapsulation, and scoping behavior within classes that evolve over time. Use them to express intent and hide mechanics.
