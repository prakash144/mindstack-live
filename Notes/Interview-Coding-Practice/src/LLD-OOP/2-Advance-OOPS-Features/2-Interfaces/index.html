    Interface¶

    🔍 Core Idea¶

    • Interface defines what a class must do — a contract — without dictating how it must do it.
    • Enables polymorphism, loose coupling, and flexibility in large-scale system design.

    ✅ When to Use¶

    • To define a contract that multiple classes can implement.
    • To achieve multiple inheritance in Java (not possible with classes).
    • To write pluggable, testable, and extensible code.

    🧰 Real-World Java Example¶

    interface PaymentProcessor {
    void processPayment(double amount);
}

class StripeProcessor implements PaymentProcessor {
    public void processPayment(double amount) {
        System.out.println("Stripe processed: $" + amount);
    }
}

class PayPalProcessor implements PaymentProcessor {
    public void processPayment(double amount) {
        System.out.println("PayPal processed: $" + amount);
    }
}

    ⚙️ Key Interface Properties¶

    Feature Supported? Notes
    Instance variables ❌ Only public static final constants allowed
    Constructors ❌ Interfaces cannot be instantiated directly
    Multiple inheritance ✅ Classes can implement multiple interfaces
    Static methods ✅ Called via interface name, not object
    Default methods (Java 8+) ✅ Allows implementation without breaking contracts

    📌 Interface Inheritance¶

    • Interfaces can extend other interfaces.
    interface Engine {
    void start();
}

interface TurboEngine extends Engine {
    void boost();
}

    📦 Static Methods in Interfaces¶

    • Belong to the interface, not instances.
    • Cannot access instance-level data.
    interface Logger {
    static void log(String message) {
        System.out.println("[LOG]: " + message);
    }
}

Logger.log("Static call");

    🧩 Default Methods - Java 8+¶

    • Add behavior to interfaces without breaking old code.
    • Can be overridden in implementing classes.
    interface Vehicle {
    default void start() {
        System.out.println("Vehicle started");
    }
}

class Car implements Vehicle {
    // inherits start() or can override it
}

    🔄 Use Cases for Default Methods¶

    Scenario Why It's Useful
    Library evolution Add methods to interfaces without breaking clients
    Multiple interfaces Avoid duplication and resolve method conflicts

    ❓ Common Interview Q\&A¶

    Q: Can interface have instance variables?

    • ❌ No. All fields are implicitly public static final. They're constants.

    Q: Can interfaces have constructors?

    • ❌ No. Interfaces have no state and cannot be instantiated.

    Q: Can a class implement multiple interfaces?

    • ✅ Yes. Enables behavior composition in a clean, modular way.

    Q: Why were default methods introduced in Java 8?

    • To allow safe API evolution.
    • Add method implementations without affecting existing classes.

    Use interfaces to write decoupled code. Program to the interface, not the implementation — it makes your system flexible and easier to test.
