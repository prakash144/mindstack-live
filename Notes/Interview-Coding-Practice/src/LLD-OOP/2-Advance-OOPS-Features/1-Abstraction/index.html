    Abstraction¶

    üîç Core Idea¶

    • Abstraction hides internal implementation, exposing only essential behavior.
    • Design Goal: Focus on what an object does, not how it does it.

    ‚úÖ When to Use¶

    • Designing APIs or SDKs where implementation may change.
    • Separating high-level policy from low-level details (e.g., Controller vs Service layer).
    • Allowing multiple implementations with a consistent interface (Strategy, Factory patterns).

    ‚öôÔ∏è Tools for Abstraction in Java¶

    • Abstract Class: Partial abstraction (can have both implemented & abstract methods)
    • Interface: Full abstraction (Java 8+ supports default & static methods)

    üß∞ Design-Level Usage¶

    Tool Use When Trade-Offs
    Abstract Class Need base logic + extensibility Single inheritance only
    Interface Enforce contract; multiple types supported No state allowed (except constants)

    üî® Real-World Java Example¶

    // Abstract Class Example
abstract class Payment {
    public abstract void makePayment();

    public void printReceipt() {
        System.out.println("Receipt printed.");
    }
}

class CreditCardPayment extends Payment {
    public void makePayment() {
        System.out.println("Paid using credit card.");
    }
}

// Interface Example
interface Notification {
    void notifyUser();
}

class EmailNotification implements Notification {
    public void notifyUser() {
        System.out.println("Sending Email notification");
    }
}

    üìå Abstract Class Key Points¶

    • Cannot be instantiated.

    • Can have:

      • Abstract methods (no body)
      • Concrete methods (with body)
      • Constructors
      • Member variables

    Constructor Example:

    abstract class Vehicle {
    Vehicle() {
        System.out.println("Vehicle constructor called");
    }
}

class Bike extends Vehicle {
    Bike() {
        System.out.println("Bike constructor called");
    }
}

    üìå Interface Key Points (Java 8+)¶

    • Default Method: Adds backward-compatible method implementation.
    • Static Method: Belongs to interface; accessed via InterfaceName.
    interface Logger {
    default void log(String message) {
        System.out.println("Default log: " + message);
    }

    static void logError(String message) {
        System.err.println("Error: " + message);
    }
}

class FileLogger implements Logger {
    // Can override log if needed
}

    üîÑ Use Case: Default Methods¶

    Purpose Example
    Backward Compatibility Add methods to existing API without breaking clients
    Multiple Interface Conflicts Provide base implementation to avoid ambiguity

    ‚ùì Common Interview Q\&A¶

    Q: Can an abstract class extend another abstract class?

    • ‚úÖ Yes. It inherits abstract and concrete methods.
    • If subclass is still abstract, it need not implement inherited abstract methods.

    Q: Can an abstract class have a constructor?

    • ‚úÖ Yes, but can't instantiate it directly.
    • Constructor runs when subclass is instantiated.
