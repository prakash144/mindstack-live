    🎥 Design a Movie Ticket Booking System¶

    🛠️ Overview¶

    An online movie ticket booking system facilitates the purchasing of movie tickets by customers.
    E-ticketing systems allow customers to browse through movies currently playing and book seats, anywhere and anytime.

    ✅ Requirements and Goals¶

    The ticket booking service should meet the following requirements:

    • 🏙️ City Listing: List the cities where affiliate cinemas are located.
    • 🎞️ Cinemas and Halls: Each cinema can have multiple halls, and each hall can run one movie show at a time.
    • ⏱️ Multiple Shows: Each movie will have multiple shows.
    • 🔎 Search Functionality: Customers can search movies by:
    • Title
    • Language
    • Genre
    • Release date
    • City name
    • 🎟️ Show and Booking:
    • Display cinemas running the selected movie and its available shows.
    • Let customers select a show at a particular cinema and book tickets.
    • 💺 Seat Selection:
    • Show the seating arrangement of the cinema hall.
    • Customers can select multiple seats based on preference.
    • Distinguish between available and booked seats.
    • 🔔 Notifications:
    • Send notifications for new movies, bookings, and cancellations.
    • 💳 Payments:
    • Customers can pay with credit cards or cash.
    • Add discount coupons during payments.
    • 🚫 Concurrency Control:
    • Ensure no two customers can reserve the same seat.

    🎭 Use Case Diagram¶

    🛡️ Actors in the System:¶

    • 👨‍💻 Admin:
    • Add/remove movies and shows.
    • Cancel movies or shows.
    • Block/unblock customers.
    • 💼 FrontDeskOfficer:
    • Book/cancel tickets.
    • 🧑‍🎓 Customer:
    • View movie schedules.
    • Book and cancel tickets.
    • 👤 Guest:
    • Search movies (but must register to book seats).
    • ⚙️ System:
    • Send notifications for new movies, bookings, and cancellations.

    📌 Top Use Cases:¶

    • 🔎 Search Movies: By title, genre, language, release date, and city name.
    • 🎟️ Create/Modify/View Booking: Book a show ticket, cancel it, or view details.
    • 💳 Make Payment: Pay for the booking.
    • 🎫 Add Coupon: Apply a discount coupon during payment.
    • 💺 Assign Seat: Display a seat map for customers to select seats.
    • 💰 Refund Payment: Refund the amount if the booking is canceled within the allowed timeframe.

    📄 Class Diagram¶

    📚 Main Classes:¶

    • 👤 Account:
    • Admin: Add/remove movies and shows, block/unblock accounts.
    • Customers: Search movies and make bookings.
    • FrontDeskOfficer: Book tickets for movie shows.
    • 👥 Guest:
    • Search and view movie descriptions.
    • Must register to make bookings.
    • 🎥 Cinema:
    • The organization for which the software is designed.
    • Attributes: name, location, etc.
    • 🏢 CinemaHall:
    • Each cinema has multiple halls with multiple seats.
    • 🏙️ City:
    • Each city can have multiple cinemas.
    • 🎬 Movie:
    • Attributes: title, description, language, genre, release date, city name.
    • ⏱️ Show:
    • Each movie can have many shows played in different halls.
    • 💺 CinemaHallSeat:
    • Represents individual seats in a cinema hall.
    • 🎟️ ShowSeat:
    • Links a Show with a CinemaHallSeat.
    • Customers book against ShowSeat.
    • 📄 Booking:
    • Contains details such as:

    • 💳 Payment:
    • Handles payment transactions.
    • 🔔 Notification:
    • Sends notifications to customers.

    🔥 Activity Diagram¶

    🎟️ Make a Booking:¶

    Any customer can perform this activity.
    Steps: 1. Search for the movie. 2. Select the cinema and show. 3. Choose seats. 4. Make the payment. 5. Receive booking confirmation.

    ❌ Cancel a Booking:¶

    Steps: 1. Search for the booking. 2. Select the booking to cancel. 3. Confirm cancellation. 4. Receive refund (if eligible).

    ⚙️ Concurrency Management¶

    To prevent multiple users from booking the same seat: - ✅ Use transactions in SQL databases. - 🔒 Leverage Transaction Isolation Levels to lock rows before updating them. - Example (SQL Server):

    SET TRANSACTION ISOLATION LEVEL SERIALIZABLE;  
BEGIN TRANSACTION;  

-- Reserve three seats (IDs: 54, 55, 56) for ShowID=99  
SELECT * FROM ShowSeat  
WHERE ShowID = 99  
AND ShowSeatID IN (54, 55, 56)  
AND isReserved = 0;  

-- If the number of rows returned is NOT three, return failure.  
UPDATE ShowSeat ...  
UPDATE Booking ...  

COMMIT TRANSACTION;
    • 🔒Serializable isolation level ensures safety from Dirty, Nonrepeatable, and Phantom Reads.

    ✅ Here is the sample Java code:¶

    import java.sql.DriverManager;
import java.sql.Connection;
import java.sql.PreparedStatement;
import java.sql.SQLException;
import java.sql.ResultSet;

public class Customer extends Person {

  public boolean makeBooking(Booking booking) {
    List<ShowSeat> seats = booking.getSeats();
    Integer seatIds[] = new Integer[seats.size()];
    int index = 0;
    for(ShowSeat seat : seats) {
      seatIds[index++] = seat.getShowSeatId();
    }

    Connection dbConnection = null;
    try {
      dbConnection = getDBConnection();
      dbConnection.setAutoCommit(false);
      // ‘Serializable’ is the highest isolation level and guarantees safety from
      // Dirty, Nonrepeatable, and Phantoms reads
      dbConnection.setTransactionIsolation(Connection.TRANSACTION_SERIALIZABLE);

      Statement st = dbConnection.createStatement();
      String selectSQL = "Select * From ShowSeat where ShowID=? && ShowSeatID in (?) && isReserved=0";
      PreparedStatement preparedStatement = dbConnection.prepareStatement(selectSQL);
      preparedStatement.setInt(1, booking.getShow().getShowId());
      Array array = dbConnection.createArrayOf("INTEGER", seatIds);
      preparedStatement.setArray(2, array);

      ResultSet rs = preparedStatement.executeQuery();
      // With TRANSACTION_SERIALIZABLE all the read rows will have the write lock, so we can
      // safely assume that no one else is modifying them.
      if (rs.next()) {
        rs.last(); // move to the last row, to calculate the row count
        int rowCount = rs.getRow();
        // check if we have expected number of rows, if not, this means another process is
        // trying to process at least one of the same row, if that is the case we
        // should not process this booking.
        if(rowCount == seats.size()) {
          // update ShowSeat table...
          // update Booking table ...
          dbConnection.commit();
          return true;
        }
      }
    } catch (SQLException e) {
      dbConnection.rollback();
      System.out.println(e.getMessage());
    }
    return false;
  }
}
