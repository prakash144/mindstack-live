    📌 Redis - Must-Know Concepts for System Design & Interviews¶

    🚀 Overview¶

    Redis (Remote Dictionary Server) is an in-memory key-value store used for high-performance applications. It supports multiple data structures and provides functionalities beyond caching, such as real-time analytics, message queues, and distributed locks.

    🔥 Why Redis?¶

    • Super Fast ⚡ - Operates in memory, offering microsecond latency.
    • Persistence 🛠️ - Supports AOF (Append Only File) and RDB (Redis Database Backup) for durability.
    • Scalability 📈 - Easily scalable via clustering and sharding.
    • Rich Data Structures 📊 - Strings, Lists, Sets, Hashes, Sorted Sets, and more.
    • Pub/Sub Messaging 📬 - Supports event-driven architecture.

    🎯 Key Use Cases & Real-Life Applications¶

    1️⃣ Caching 🏷️ (e.g., URL Shortener, Instagram Feeds)¶

    • Reduces database load by storing frequently accessed data.
    • Example: URL Shortener 📎
      • Short URL → Redis (Cache) → Redirect to actual URL.
      • Avoids repeated database lookups for the same URL.

    2️⃣ Session Management 🔑 (e.g., User Authentication, E-commerce Websites)¶

    • Stores user session data (cookies, JWT tokens) for quick access.
    • Example: E-commerce Websites (Amazon, Flipkart) 🛒
      • Stores active cart items for fast retrieval.
      • Reduces load on relational databases.

    3️⃣ Concurrency Management 🔄 (e.g., BookMyShow, Ticket Booking Systems)¶

    • Prevents race conditions using Timer-Based Locking.
    • Example: BookMyShow 🎟️
      • When a user selects a seat, Redis locks it with a TTL (Time-To-Live).
      • If the transaction is incomplete, the lock expires, making the seat available again.

    4️⃣ Rate Limiting ⏳ (e.g., API Throttling, Login Attempts)¶

    • Controls the number of requests a user can make in a time frame.
    • Example: Login Protection (Brute Force Prevention) 🔐
      • Allows 5 failed login attempts per minute.
      • Blocks further attempts for a certain duration.

    5️⃣ Message Queue & Pub/Sub 📢 (e.g., Chat Applications, Real-time Notifications)¶

    • Acts as a lightweight message broker for async communication.
    • Example: WhatsApp, Slack 💬
      • Sends real-time notifications and chat messages via Redis Pub/Sub.

    6️⃣ Leaderboards & Counting 🏆 (e.g., Gaming Apps, YouTube Views)¶

    • Uses Sorted Sets for ranking and real-time counters.
    • Example: Online Multiplayer Games (PUBG, Fortnite) 🎮
      • Stores player scores and retrieves top-ranked players instantly.

    7️⃣ Distributed Locking 🔓 (e.g., Avoiding Double Spending, Inventory Management)¶

    • Prevents multiple processes from modifying the same resource simultaneously.
    • Example: Payment Gateways (UPI, Paytm, Stripe) 💰
      • Ensures a transaction is processed only once.
      • Uses RedLock Algorithm to avoid inconsistencies.

    8️⃣ Full-Page Caching 🌍 (e.g., CDN, Reverse Proxy)¶

    • Caches entire webpages to reduce load times.
    • Example: Cloudflare, Fastly, Nginx 🏎️
      • Serves pre-rendered content, reducing server load.

    ⚙️ Important Redis Commands¶

    Command Description
    SET key value Store a value under a key
    GET key Retrieve a value by key
    EXPIRE key seconds Set TTL for a key
    INCR key Increment a key’s value
    DECR key Decrement a key’s value
    HSET key field value Store a value in a hash field
    HGET key field Retrieve a value from a hash field
    ZADD key score member Add a member to a sorted set
    ZRANGE key start stop Retrieve members from a sorted set

    🔍 Interview Tips & Best Practices¶

    • ✅ Use Redis for Read-Heavy Workloads: Cache database queries to improve performance.
    • ✅ Use Expiration (EXPIRE) for Caching: Prevent stale data from persisting indefinitely.
    • ✅ Avoid Using Redis as a Primary Database: It lacks full ACID compliance.
    • ✅ Use Lua Scripting for Atomic Operations: Prevents race conditions in concurrent writes.
    • ✅ Monitor Performance with redis-cli: Use INFO command to check memory usage and stats.
    • ✅ Use Redis Clustering for High Availability: Distribute data across multiple nodes.
    • ✅ Apply Least Recently Used (LRU) Eviction: Use maxmemory-policy allkeys-lru to optimize caching.

    🎤 Conclusion¶

    Redis is a powerful tool in system design, offering solutions for caching, concurrency, real-time analytics, and more. Understanding its data structures, use cases, and best practices is crucial for interviews and building high-performance applications.

    🔹 Key Takeaway: If your system is performance-sensitive and requires fast reads, session storage, or real-time data, Redis is your go-to solution!

    🚀 Keep practicing, and happy coding! 🎯
